# ` <sort>{:dn}`

` <sort>{:dn}` is an [**General Operator**](../reference/essentialConcepts/#componentTypes) component that reorders a list of components based on given criteria.

## Attributes and Properties 
 <AttrDisplay name='sort'/> 
 <PropDisplay name='sort'/>

---

### Example: ` <sort>{:dn}` a list of numbers


```doenet-editor-horiz
<p>
  <asList>
    <sort>
      <number>4</number>
      <number>pi</number>
      <number>sqrt(3)</number>
      <number>11</number>
  </sort>
  </asList>
</p>
```



In this example, ` <number>{:dn}` component are sorted in increasing order using the ` <sort>{:dn}` component.

### Example: sortByComponent


```doenet-editor-horiz
Drag the points!
<graph>
  <point name="A">(0,5)</point>
  <point name="B" styleNumber="2">(-2,6)</point>
  <point name="C" styleNumber="3">(7,-3)</point>
  <point name="D" styleNumber="4">(-4,2)</point>
</graph>
<p>Choose component number to sort by (1 or 2):
  <mathInput name="i" prefill="1"/></p>
<p>
  <asList>
    <sort sortByComponent="$i">
      $A $B $C $D
    </sort>
  </asList>
</p>
```



For multi-dimensional object like points, sorting by `sortByComponent="1"` sorts by their first component, i.e., their $x$-component, while using `sortByComponent="2"` results in sorting based on the $y$-component.


Values entered outside the dimensions of the object, such as entering a $3$ or a $0$ in for a two-dimensional object, will result in no sorting.

### Example: sortByProp


```doenet-editor-horiz
Drag the points!
<graph size="small">
  <point name="A">(0,5)</point>
  <point name="B">(-2,6)</point>
  <point name="C">(7,-3)</point>
  <point name="D">(3,2)</point>
  <point name="E">(5,1)</point>
</graph>
<p>Points sorted by x: 
  <sort sortByProp="x">
    $A $B $C $D $E
  </sort>
</p>
<p>Points sorted by y:
  <sort sortByProp="y">
    $A $B $C $D $E
  </sort>
</p>
```



Components with numerical properties can be sorted based on an individual property. In this example, the points are sorted in ascending order based on their $x$-coordinates, and then by their $y$-coordinates.

### Example: sortVectorsBy


```doenet-editor-horiz
Drag the vectors!
<graph size="small">
  <vector name="v1" styleNumber="1">(0,6)</vector>
  <vector name="v2" tail="(2,2)" displacement="(3,7)" styleNumber="2"/>
  <vector name="v3" tail="(7,1)" displacement="(-1,-3)" styleNumber="3"/>
  <vector name="v4" tail="(-2,-2)" displacement="(-3,0)" styleNumber="4"/>
</graph>
<p>Vectors sorted by tail:
  <asList>
    <sort sortVectorsBy="tail" >
      $v1 $v2 $v3 $v4
    </sort>
  </asList>
</p>
<p>Vectors sorted by displacement:
  <asList>
    <sort sortVectorsBy="displacement" sortByProp="x2">
      $v1 $v2 $v3 $v4
    </sort>
  </asList>
</p>
```



Vectors can be sorted based on the location of their `tail` or by their `displacement`.


When sorted based on `tail`, Doenet defaults to sorting based on the $x$-coordinate unless specified otherwise using `sortByProp="y"`.


When sorted based on `displacement`, Doenet defaults to sorting based on the displacement in the $x$-direction unless specified otherwise using `sortByProp="x2"`, as in this example. This is because vector displacements are individually accessed using the convention: `x1` for the horizontal displacement, and `x2` for the vertical displacement.
