# ` <sources>{:dn}`





The ` <sources>{:dn}` component is used within a [` <map>{:dn}`](map.mdx) block to define the values to which the [` <template>{:dn}`](template.mdx) will be applied.  The ` <sources>{:dn}` component may enclose any combination of components; the ` <map>{:dn}` will iterate over the final result of components inside the ` <sources>{:dn}`.  For example, a ` <sources>{:dn}` could contain a <ref uri="doenet:activityId=___VgxVIdpE3sN4POU0EKT">` <sequence/>{:dn}`</ref>, a series of user-inputs, or other numerical or text values.


---

### Example: A minimal map


```doenet-editor-horiz
<asList>
  <map>
    <template><math simplify>$num*2</math></template>
    <sources alias="num">
      <number>1</number>
      <number>12</number>
      <number>576</number>
    </sources>
  </map>
</asList>
```



In this example, the ` <sources>{:dn}` component is used with a list of arbitrary <ref>` <number>{:dn}`</ref> components. The [` <map>{:dn}`](map.mdx) will apply the [` <template>{:dn}`](template.mdx) to each instance of the data contained within the ` <sources>{:dn}` component in turn.


The `alias` attribute specifies the identifier that will be applied to each of the individual components inside ` <sources>{:dn}` as the mapping is performed. Since `alias` functions similarly to a `name`, it is referenced by the ` <template>{:dn}` using the `$name` notation. It cannot, however, be referenced outside the context of the ` <map>{:dn}`.



---

### Example: A map with multiple sources


```doenet-editor-horiz
<graph xmin="-1" xmax="5" ymin="-1" ymax="5" size="small">
  <map behavior="combination">
    <template>
      <point>($i, $j)</point>
    </template>
    <sources alias="i">
      <sequence from="1" to="4"/>
    </sources>
    <sources alias="j">
      <sequence from="1" to="4"/>
    </sources>
  </map>
</graph>
<graph xmin="-1" xmax="5" ymin="-1" ymax="5" size="small">
  <map behavior="parallel">
    <template>
      <point>($i, $j)</point>
    </template>
    <sources alias="i">
      <sequence from="1" to="4"/>
    </sources>
    <sources alias="j">
      <sequence from="1" to="4"/>
    </sources>
  </map>
</graph>
```



Multiple sources can be used. How the sources are combined can be specified in the parent ` <map>{:dn}` using the `behavior` attribute. The default behavior, shown in the first graph, is `combination`. The second graph illustrates the `parallel` behavior.

### Example: alias


```doenet-editor-horiz
<graph ymin="0" ymax="14" size="small">
  <map>
    <template>
      <line styleNumber="$i">y = 2 $i</line>
    </template>
    <sources alias="i"><sequence from="1" to="6"/></sources>
  </map>
</graph>
```



The `alias` attribute specifies the identifier that will be applied to each of the individual components inside ` <sources>{:dn}` as the mapping is performed. Since `alias` functions similarly to a `name`, it is referenced by the ` <template>{:dn}` using the `$name` notation. It cannot, however, be referenced outside the context of the ` <map>{:dn}`.



---

### Example: indexAlias


```doenet-editor-horiz
<p>
  <map>
    <template>Plant $i is a $plant. </template>
    <sources alias="plant" indexAlias="i">
      <text>tree</text>
      <text>flower</text>
      <text>cactus</text>
    </sources>
  </map>
</p>
```



In this example, the `alias` attribute of ` <sources>{:dn}` refers to the components inside ` <sources>{:dn}` while the `indexAlias` attribute refers to the iteration number as the ` <map>{:dn}` iterates throught the sources.  Both aliases are referenced by the [` <template>{:dn}`](template.mdx) component using the `$name` notation.


Note the space included at the end of the ` <template>{:dn}` needed to include spaces between the sentences when rendered sequentially.
