# `<abs>`

## What it does
The `<abs>` component renders the absolute value of the enclosed `math`. It belongs to a class of components known as [**Math Operators**](../componentClasses). (Several other components in this category include `<sum>`, `<product>`, `<ceil>` and `<round>`.)

Using the `<abs>` component is equivalent to using the `|` symbol to enclose an expression within a `<math>` or `math` type component or attribute specification. 

## Basic usage examples

### Example: Render the absolute value of a math expression.
<DoenetML>
{`
<codeEditor showResults resultsLocation="left">
    <m>|5 - 8| = <abs>5 - 8</abs></m>
</codeEditor>
`}
</DoenetML>


### Example: Absolute value of `<mathInput>`
<DoenetML>
{`
<codeEditor showResults resultsLocation="left">
    <p>Type any number: <mathInput name="userN"/></p>
    <p><m>|$userN| = <abs>$userN</abs></m></p>
</codeEditor>
`}
</DoenetML>

### Example: Absolute value of numerical properties
<DoenetML>
{`
<codeEditor showResults resultsLocation="bottom" height="600">
    <p>Move the point in the third quadrant.</p>
    <graph size="small" name="g1" newNameSpace>
        <point name="p1" styleNumber="2">(-5,-5)</point>
        <point name="p2" draggable="false">
            (<abs>$p1.x</abs>, <abs>$p1.y</abs>)
        </point>
    </graph>
</codeEditor>
`}
</DoenetML>

Children of the `<abs>` component can include references to `<mathInput>` values or other `math` quantities, such as single-valued numerical properties of graphical objects. Invalid children are strings and tuples. 


### Example: Absolute value of a function
<DoenetML>
{`
<codeEditor showResults resultsLocation="bottom" height="600">
    <graph size="small" name="g2" newNameSpace>
       <function name="f">(1/4)x^2 - 4</function>
       <function styleNumber="2">
         <abs>$f</abs>
       </function>
     </graph>
</codeEditor>
`}
</DoenetML>

The `<abs>` component is used to render the absolute value of a named `<function>` within a `<graph>`.

---

## Special usage notes

Note that the `<abs>` component cannot be used directly in attribute specification. You can, however, use the `|` notation within an attribute specification. You can also define a `<math>` component separately, which contains the `<abs>` component, and then reference it in an attribute specification.
