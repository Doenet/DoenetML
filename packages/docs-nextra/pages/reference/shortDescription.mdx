# `<shortDescription>{:dn}`

`<shortDescription>{:dn}` supplies a concise text alternative for an enclosing
component that is consumed **only by assistive technology**, such as screen readers.
It is never shown as visible content to sighted users.

Because it is invisible on screen, a `<shortDescription>{:dn}` **can** be used to
satisfy accessibility requirements — it provides the accessible name or description
that components like `<image>{:dn}`, `<video>{:dn}`, and `<graph>{:dn}` need. This
is the opposite of `<description>{:dn}`, which is shown to everyone in a popup and
therefore does not satisfy an accessibility requirement.

A `<shortDescription>{:dn}` is meaningful as a child of `<image>{:dn}`,
`<video>{:dn}`, `<graph>{:dn}`, `<answer>{:dn}`, `<mathInput>{:dn}`,
`<textInput>{:dn}`, `<booleanInput>{:dn}`, `<choiceInput>{:dn}`, and
`<matrixInput>{:dn}`.

How it reaches assistive technology depends on the component: it becomes the `alt`
text of an image, the `title` of a video, and an `aria-label` or `aria-labelledby`
for an input that has no visible `<label>{:dn}`. When an input already has a visible
`<label>{:dn}`, the `<shortDescription>{:dn}` is added as an `aria-description`
instead, so the two do not conflict.

Keep the text short and write it in plain language. In particular, spell out any
mathematics in words rather than nesting math components inside it — Doenet emits an
accessibility warning if a `<shortDescription>{:dn}` contains math, since a screen
reader cannot reliably read raw math notation.

---

### Example: `<shortDescription>{:dn}` on an `<image>{:dn}`


```doenet-editor-horiz
<image source="https://upload.wikimedia.org/wikipedia/commons/1/16/DoricParthenon.jpg">
  <shortDescription>A corner of a Doric temple with column capital, architrave, and triglyph</shortDescription>
</image>
```



Every `<image>{:dn}` should have a `<shortDescription>{:dn}` so that screen-reader
users receive a meaningful text alternative. Its text becomes the image's `alt` text
and is not displayed on screen.



---

### Example: `<shortDescription>{:dn}` on a `<graph>{:dn}`


```doenet-editor-horiz
<graph>
  <shortDescription>A graph of the line y equals x</shortDescription>
  <function>x</function>
</graph>
```



A `<graph>{:dn}` is a visual component, so a `<shortDescription>{:dn}` gives
assistive technology a summary of what the graph displays.



---

### Example: `<shortDescription>{:dn}` on a `<mathInput>{:dn}`


```doenet-example
<p>Enter the derivative of <m>x^2</m>:
  <mathInput name="deriv">
    <shortDescription>derivative of x squared</shortDescription>
  </mathInput>
</p>
```



When an input has no visible `<label>{:dn}`, its `<shortDescription>{:dn}` provides
the accessible name screen-reader users need to identify the control. Note that the
mathematics is spelled out in words ("x squared") rather than written with a
`<m>{:dn}` component.
