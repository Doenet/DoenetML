# `<problems>{:dn}`


`<problems>{:dn}` is a [Sectional](../document_structure/essentialConcepts#component-types)
component that groups a collection of [`<problem>{:dn}`](problem)
components (and other content) and, by default, renders the contained
sectional components as a numbered list. It behaves like
[`<section>{:dn}`](section) in other respects.


<AttrPropDisplay name='problems'/>


---

### Example: `<problems>{:dn}` rendering its children as a list


```doenet-editor-horiz
<problems>
  <title>Practice problems</title>

  <problem>
    <p>What is <m>2 + 2</m>? <answer>4</answer></p>
  </problem>

  <problem>
    <p>What is <m>3 \times 5</m>? <answer>15</answer></p>
  </problem>

  <problem>
    <p>What is <m>12 - 7</m>? <answer>5</answer></p>
  </problem>
</problems>
```



`<problems>{:dn}` sets `asList` to `true` by default, so each child
section is rendered as a numbered list item.



---

### Example: `<problems>{:dn}` with default title


```doenet-editor-horiz
<problems>
  <problem><p><answer>1</answer></p></problem>
  <problem><p><answer>2</answer></p></problem>
</problems>
```



When `<problems>{:dn}` has no `<title>{:dn}`, an auto-generated title
(e.g. `Problems 1`) is rendered.



---

### Attribute Example: asList


```doenet-example
<problems asList="false">
  <title>Problems without list rendering</title>

  <problem><title>Problem 1</title>
    <p><answer>1</answer></p>
  </problem>
  <problem><title>Problem 2</title>
    <p><answer>2</answer></p>
  </problem>
</problems>
```



Set `asList="false"` to render the contained problems as ordinary
sectional blocks rather than as list items.



---

### Attribute Example: hide


```doenet-example
<booleanInput name="hideProblems">
  <label>Hide the problem set</label>
</booleanInput>

<problems hide="$hideProblems">
  <title>Problem set</title>
  <problem><p>Type "sassafras": <answer type="text">sassafras</answer></p></problem>
</problems>
```



The `hide` attribute takes a boolean as input and can be used with any rendered component.
