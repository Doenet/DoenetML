# ` <moduleAttributes>{:dn}`

` <moduleAttributes>{:dn}` is a [General Operator](../document_structure/essentialConcepts#component-types)
component that is used to describe custom user-defined attributes when nested in a ` <module>{:dn}` component. 
Similar to the attributes of a built-in DoenetML component, the user-defined attributes defined with a ` <moduleAttributes>{:dn}` block
are then listed in the opening tag of the custom ` <module>{:dn}` component.

### Example: ` <module>{:dn}` for repeated geometry


```doenet-editor-horiz
<setup>
<module name="drawBalloon">
  <moduleAttributes>
     <point name="center">(0,2)</point>
     <number name="color">2</number>
     <number name="radius">4</number>
  </moduleAttributes>
  
  <lineSegment 
    endpoints="($center.x, $center.y - $radius) (1,-7)" 
    styleNumber="5"
  />
  <circle 
    center="$center"
    radius="$radius"
    styleNumber="$color"
    filled
  />
</module>
</setup>

<graph 
    ymin="-8" 
    ymax="8" 
    displayXAxis="false" 
    displayYAxis="false">
  <module extend="$drawBalloon"/>
  <module 
    extend="$drawBalloon" 
    center="(-5,0)" 
    color="1"
  />
  <module 
    extend="$drawBalloon" 
    center="(5,1)" 
    color="3" 
    radius="3"
  />
</graph>
```

In this example, a ` <module>{:dn}` named `drawBalloon` is defined within a `<setup>{:dn}` block. Nested 
`<moduleAttributes>{:dn}` components define the center point, color, and radius of a default balloon. 

Geometry defined within the ` <module>{:dn}` (the `<lineSegment>{:dn}` and `<circle>{:dn}` components) 
are copied into the graph when the module is referenced.

Referencing the module multiple times within the `<graph>{:dn}` and varying the
attribute values has the effect of creating a "bunch" of balloons.
