# ` <number>{:dn}`


` <number>{:dn}` stores the numerical value of the enclosed argument, and renders a numerical approximation of the argument to a specified precision.


The default precision for display is either: (1) rounded to three significant digits or (2) rounded two decimal places, whichever has more digits.

## Attributes and Properties

### Example: Display numerical approximation


```doenet-editor-horiz
<p><number>pi</number></p>
<p><number>1/sqrt(2)</number></p>
```



The default precision for display is either (1)rounded to three significant digits or (2) rounded two decimal places, whichever has more digits. 



---

### Example: renderAsMath


```doenet-editor-horiz
<p><math>pi</math></p>
<p><number>pi</number></p>
<p><number renderAsMath>pi</number></p>
```



The `renderAsMath` attribute adjusts the default font for a ` <number>{:dn}` in order for it to appear the same as that of a ` <math>{:dn}`.



---

### Example: convertBoolean


```doenet-editor-horiz
<p><number convertBoolean>2 = 1 + 1</number></p>
<p><number convertBoolean>1 + 1 = 1</number></p>      
```



The `convertBoolean` attribute renders or stores a boolean that evaluates to `true` as 1 and to `false` as 0.



---

### Example: valueOnNaN


```doenet-editor-horiz
<p>Enter a number other than 0: <mathInput name="userN"/></p>
<p><number>$userN</number></p>
<p><number valueOnNaN="0" name="numN">$userN</number></p>
<p hide="$numN = 0">You've entered a number</p>
```



The `valueOnNaN` attribute allows for temporary values (which can be used in boolean conditions) to be stored by Doenet until a new number is assigned

The `anchor` attribute sets the location of a ` <number>{:dn}` placed within a ` <graph>{:dn}`.

### Example: draggable


```doenet-editor-horiz
<p>Try dragging each number.</p>
<graph size="small">
  <number name="n1" anchor="(5,5)">53</number>
  <number name="n2" draggable="false" anchor="(-5,-5)">21</number>
</graph>      
<p>53 is <c>draggable</c> = $n1.draggable</p>
<p>21 is <c>draggable</c> = $n2.draggable</p>     
```



The `draggable` attribute determines whether the ` <point>{:dn}` can be dragged around the ` <graph>{:dn}`. All graphical objects are draggable by default.

### Example: displayDigits/displayDecimals


```doenet-editor-horiz
<p>By default, display three digits or two decimals places, whichever is greater.</p>
<p><m>\frac{8}{7} \approx </m> 
  <number>8/7</number></p>
<p><m>\frac{2}{3} \approx </m>
  <number>2/3</number></p>
<p>User specifies number of digits to display; default number of decimal places is ignored</p>
<p><m>\frac{8}{7} \approx </m>
  <number displayDigits="2">8/7</number></p>
<p>User specifies number of decimal places to display; default number of digits is ignored</p>
<p><m>\frac{2}{3} \approx </m>
  <number displayDecimals="2">2/3</number></p>
```



By default, numbers will be displayed as rounded to three digits or to two decimal places, whichever includes more digits.  The `displayDigits` attribute will change the number of digits, and the `displayDecimals` attribute will change the number of decimal places.


Note: these display rounding settings do not affect the actual value of the numbers in the expression.  All available digits will be used if the expression is used in another calculation unless one uses the ` <round>{:dn}` component to explicitly round numbers.

### Example: displaySmallAsZero


```doenet-editor-horiz
<p>The exact number is: <m>1 \times 10^{-21}</m></p>
<p>By default, small numbers (with absolute value smaller than <m>10^{-14}</m>) are displayed as zero.</p>
<number name="n1">10^(-21)</number>
<p>Don't display small numbers as zero.</p>
<number name="n2" displaySmallAsZero="false" >10^(-21)</number>
<p>Display numbers smaller than <m>10^{-20}</m> in absolute value as zero.</p>
<p><number name="n3" displaySmallAsZero="10^(-20)">10^(-19)</number></p>
<p><number name="n4" displaySmallAsZero="10^(-20)">10^(-21)</number></p>
```



The default value of the `displaySmallAsZero` attribute is $10^{-14}$, which means that numerical values smaller than that value will appear as zero.  This behavior can be altered by changing the value of `displaySmallAsZero` to a different number, or even setting it to "false".



---

### Example: padZeros


```doenet-editor-horiz
<p><number displayDecimals="2">1/10</number></p>
<p><number displayDecimals="2" padZeros>1/10</number></p>
```



If the `padZeros` is set, then numbers will be displayed padded with zeros so that all the digits from the combination of `displayDigits` and `displayDecimals` will be shown.
