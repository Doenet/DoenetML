# ` <boolean>{:dn}`


` <boolean>{:dn}` is a [Logic](../document_structure/essentialConcepts#component-types) 
component that renders the boolean value of `true` or `false` of the enclosed expression.

## Attributes and Properties 
 <AttrDisplay name='boolean'/> 
 <PropDisplay name='boolean'/>


---

### Example: ` <boolean>{:dn}` value of an equation


```doenet-example
<p>The following equation is 
  <alert><boolean>1 + 1 = $input</boolean></alert>.</p>
  <m>1 + 1 = </m> <mathInput name="input" prefill="3"/>
```



A ` <boolean>{:dn}` component is used to render whether an 
equation is `true` or `false` for a user-supplied response



---

### Example: ` <boolean>{:dn}` value of a geometric property


```doenet-editor-horiz
<graph size="small">
    <point name="p1">(-4,-4)</point>
    <point name="p2">(4,4)</point>
    <line name="line1" through="$p1 $p2"/>
  </graph>

  <p>The following statement is <alert><boolean>$line1.slope >= 1</boolean></alert>.</p>
  <p><em>The slope of the line is greater than or equal to <m>1</m>.</em></p>
```



A ` <boolean>{:dn}` component is used to render whether a statement regarding 
the geometric slope of a named ` <line>{:dn}` is `true` or `false`. Adjust the slope of 
the line to affect the ` <boolean>{:dn}`.

### Example: Constructing compound ` <boolean>{:dn}` expressions


```doenet-editor-horiz
 <p>I'm thinking of a number that is less than 100, 
 greater than 20, and is divisible by 3. 
 What could my number be? <mathInput name="userNumber"/></p>

<p>Is it less than 100? 
  <boolean name="lessThan100">
    $userNumber < 100
  </boolean>
</p>

<p>Is it greater than 20? 
  <boolean name="greaterThan20">
    $userNumber > 20
  </boolean>
</p>

<p>How about divisible by 3? 
  <boolean name="divisibleBy3">
    <mod>$userNumber 3</mod> = 0
  </boolean>
</p>

<p>$userNumber could be the number... <alert>
  <boolean>
    $lessThan100 = true and 
    $greaterThan20 = true and 
    $divisibleBy3 = true
  </boolean>
  !</alert>

```



Multiple named ` <boolean>{:dn}` components can be combined to create 
more complex boolean expressions.

### Attribute Example: symbolicEquality


```doenet-example
<p>Enter <m>x^2 + 2x + 1</m> exactly: 
  <mathInput name="mi"/></p>
  <boolean symbolicEquality>$mi = x^2 + 2x + 1</boolean>
```



Comparing two values with the `symbolicEquality` attribute switches demands exact syntactical equality.



---

### Attribute Example: expandOnCompare


```doenet-example
<p>Enter <m>x^2 + 2x + 1</m> in factored form: 
  <mathInput name="mi"/></p>
  <boolean expandOnCompare symbolicEquality>$mi = x^2 + 2x + 1</boolean>
```



Comparing two values with the `expandOnCompare` and `symbolicEquality` attributes 
allows expressions that are equivalent after expanding to evaluate to `true`.

### Attribute Example: unorderedCompare


```doenet-editor-horiz
<p>Enter <m>1,2,3,4,5</m> in that exact order: 
  <mathInput name="mi"/></p>

<p><boolean>
  $mi = <mathList>1 2 3 4 5</mathList>
</boolean></p>

<p>Enter <m>1,2,3,4,5</m> in mixed-up order: 
  <mathInput name="mi2"/></p>

<p><boolean unorderedCompare>
  $mi2 = <mathList>1 2 3 4 5</mathList>
</boolean></p>
```



When multiple values are compared within a single ` <boolean>{:dn}` tag, 
the default behavior is to match the order of the sequenced values. 
If order is irrelevant, use the `unorderedCompare` attribute.



---

### Attribute Example: matchByExactPositions


```doenet-editor-horiz
<p>Enter <m>1,2,3</m> in that exact order: </p>
  <mathInput name="mi1"/>, 
  <mathInput name="mi2"/>, 
  <mathInput name="mi3"/>
<p><boolean matchByExactPositions>
  ($mi1, $mi2, $mi3) = (1, 2, 3)
</boolean></p>

<p>Now, observe what happens when you switch the order</p>
```



When multiple responses are evaluated by comparison with a 
vector of correct values inside a ` <boolean>{:dn}`, the `matchByExactPositions` attribute 
enforces a strict order onto the validation. Only correct values in the correct answer 
blanks will allow the ` <boolean>{:dn}` to evaluate to `true`.



---

### Attribute Example: allowedErrorInNumbers


```doenet-example
<p>Enter a number within <m>1\%</m> of <m>100</m>: 
  <mathInput name="mi1"/></p>
  <boolean allowedErrorInNumbers="0.01">$mi1 = 100</boolean>

<p>Enter a number within <m>10\%</m> of <m>100</m>: 
  <mathInput name="mi2"/></p>
  <boolean allowedErrorInNumbers="0.1">$mi2 = 100</boolean>
```



The `allowedErrorInNumbers` attribute may be used to specify the 
acceptable error of an approximate response when comparing numerical values. 
By default, the error is specified as a fractional value.

---

### Attribute Example: allowedErrorIsAbsolute


```doenet-example
<p>Enter a number within <m>1</m> of <m>100</m>: 
  <mathInput name="mi1"/></p>
  <boolean allowedErrorInNumbers="1" allowedErrorIsAbsolute>
    $mi1 = 100
  </boolean>

<p>Enter a number within <m>10</m> of <m>100</m>:  
  <mathInput name="mi2"/></p>
  <boolean allowedErrorInNumbers="10" allowedErrorIsAbsolute>
    $mi2 = 100
  </boolean>
```

To specify an allowed error with a specific numerical 
value (rather than as a fraction of the correct response), use 
the `allowedErrorIsAbsolute` attribute in combination 
with the `allowedErrorInNumbers` attribute.
