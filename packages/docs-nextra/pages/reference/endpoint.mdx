# ` <endpoint>{:dn}`


` <endpoint>{:dn}` is a [Graphical](../document_structure/essentialConcepts#component-types) 
component that renders a point that can be switched from open to closed (or unfilled to filled).

## Attributes and Properties 
 <AttrDisplay name='endpoint'/> 
 <PropDisplay name='endpoint'/>


---

### Example: Default endpoint


```doenet-editor-horiz
<graph size="small" displayXAxis="false" 
  displayYAxis="false">
  <endpoint name="default" labelIsName/>  
  <endpoint name="switchable" switchable
    coords="(0,5)"  labelIsName/>
  <endpoint name="open" open 
    coords="(0,-5)" labelIsName/>
</graph>
```



The default ` <endpoint>{:dn}` behaves like a ` <point>{:dn}`. 
An ` <endpoint>{:dn}` also accepts all ` <point>{:dn}` attributes.



---

### Example: An ` <endpoint>{:dn}` shares ` <point>{:dn}` attributes


```doenet-editor-horiz
<graph size="small">
  <endpoint name="ep" x="-2" y="4"
    open="true" switchable="true"
    styleNumber="2" labelPosition="lowerleft" 
    applyStyleToLabel>
    <label><m>P = $ep.coords</m></label>
    <constraints>
      <constrainToGrid dx="2" dy="2"/>
    </constraints>
  </endpoint>
</graph>
```



All attributes and properties that are available to points are also available to 
endpoints. Endpoints can also have the same ` <constraints>{:dn}` as points.



---

### Attribute Example: switchable


```doenet-editor-horiz
<graph size="small" displayXAxis="false" 
  displayYAxis="false">
  <endpoint switchable="true">
    <label>Click me!</label>
  </endpoint>
</graph>
```



The `switchable` attribute allows the user to click the ` <endpoint>{:dn}` to 
switch between an open and a closed point.



---

### Attribute Example: open


```doenet-editor-horiz
<graph size="small" displayXAxis="false" 
  displayYAxis="false">
  <endpoint open="true"/>
</graph>
```



The `open` attribute renders an ` <endpoint>{:dn}` that is open.



---

### Property Example: ` <point>{:dn}` properties


```doenet-editor-horiz
<graph size="small">
  <endpoint name="ep" x="-2" y="4"
    open="true" switchable="true"
    styleNumber="2" labelPosition="lowerleft" 
    applyStyleToLabel>
    <label><m>P = $ep.coords</m></label>
    <constraints>
      <constrainToGrid dx="2" dy="2"/>
    </constraints>
  </endpoint>
</graph>

<p><c>coords</c> = $ep.coords</p>
<p><c>styleDescriptionWithNoun</c> 
  = $ep.styleDescriptionWithNoun</p>
<p><c>fixed</c> = $ep.fixed</p>
```



The properties of points are accessible to endpoints as well. A few selected properties are illustrated.



---

### Property Example: attributes as properties


```doenet-editor-horiz
<graph size="small">
  <endpoint name="ep" open switchable/>
</graph>

<p><c>switchable</c> = $ep.switchable</p>
<p><c>open</c> = $ep.open</p>
```



Both attributes of ` <endpoint>{:dn}` are also accessible as properties.
