# `<textInput/>{:dn}`

<ComponentDisplay name='textInput'/>

`<textInput/>{:dn}` is an [Input](../document_structure/essentialConcepts#component-types)
component that renders a form input field for text responses from the user.

<AttrPropDisplay name='textInput'/>


## Examples

### Example: Default `<textInput/>{:dn}`


```doenet-editor-horiz
<p>
  <textInput name="userText">
    <label>Type anything</label>
  </textInput>
</p>

<p>You typed: $userText</p>
```



The default `<textInput/>{:dn}` blank is wide enough for approximately 2-4 words to be visible at a time.



---

### Example: Large `<textInput/>{:dn}`


```doenet-editor-horiz
<p>
  <textInput name="longText" expanded>
    <label>Type some longer things</label>
  </textInput>
</p>

<p>You typed: $longText</p>
```



The expanded `<textInput/>{:dn}` blank is approximately large enough for 4-6 sentences to be visible at a time.


---

### Example: `<textInput/>{:dn}` with `<label>{:dn}` child


```doenet-editor-horiz
<p> 
  <textInput name="userName">
    <label>Enter your name:</label> 
  </textInput>
</p>

<p>You typed: $userName</p>
```



The `<textInput/>{:dn}` component can take a nested `<label>{:dn}` child.



---

### Example: `<textInput/>{:dn}` for graded free-response


```doenet-editor-horiz
<p>
  <answer name="ans" type="text" handGraded>
    <textInput expanded>
      <label>Explain how you solved the problem</label>
    </textInput>
  </answer>
</p>
```



The `<textInput/>{:dn}` can be nested in an `<answer>{:dn}` with the `handGraded` attribute for graded free-response style questions.



---

### Example: `<textInput/>{:dn}` for graded closed-form text response


```doenet-editor-horiz
<p>
  <textInput name="userColor">
    <label>What color is the complement of purple?</label>
  </textInput>
  <answer 
    name="ans" 
    type="text"
    caseInsensitiveMatch
  >
    <award>
      <when>
        $userColor = yellow
      </when>
    </award>
  </answer>
</p>
```



The `<textInput/>{:dn}` can be nested in an `<answer>{:dn}` that checks the entered text against an expected value, automatically grading a closed-form text response.



---

### Example: `<textInput/>{:dn}` on a `<graph>{:dn}`


```doenet-editor-horiz
<p>
  <mathInput name="userAnchor" prefill="(0,0)">
    <label>Where should we put the <tage>textInput</tage>?</label>
  </mathInput>
</p>

<graph size="small"> 
  <shortDescription>A regular pentagon with a draggable text input.</shortDescription>
  <textInput name="shape" anchor="$userAnchor">
    <shortDescription>text input anchored at $userAnchor</shortDescription>
  </textInput>
  <regularPolygon numSides="5" sideLength="6" filled center="(-2,0)"/>
</graph>
```



A `<textInput/>{:dn}` can be placed on a `<graph>{:dn}`. Use the `anchor` attribute to adjust its starting location. Click on the border of the `<textInput/>{:dn}` to drag it around.



## Attribute Examples

### Attribute Example: prefill


```doenet-example
<p>
  <textInput prefill="my cabin">
    <label>Enter your favorite vacation place</label>
  </textInput>
</p>
```



The `prefill` attribute specifies initial text for the `<textInput/>{:dn}`.



---

### Attribute Example: expanded


```doenet-example
<p>
  <textInput expanded>
    <label>Type some longer things</label>
  </textInput>
</p>
```



The `expanded` attribute renders a larger `<textInput/>{:dn}` space. The dimensions of the input space can be further customized with the `height` and `width` attributes.

---

### Attribute Example: forAnswer

```doenet-editor-horiz
<setup>
  <textList name="validColors">red orange yellow green blue indigo violet</textList>
</setup>
<p>Enter two colors of ROYGBIV.
  <textInput name="c1" forAnswer="$ans">
    <shortDescription>first color</shortDescription>
  </textInput>
  <textInput name="c2" forAnswer="$ans">
    <shortDescription>second color</shortDescription>
  </textInput>
  <answer name="ans" matchPartial caseInsensitiveMatch>
    <award><when>$c1 elementof $validColors and $c2 elementof $validColors</when></award>
  </answer>
</p>

<p>Submitted responses: $ans.submittedResponses</p>

```

Specifying the attribute `forAnswer` of a `<textInput>` does two things:
1. It colors the border of the text input based on the correctness of the response (assuming `colorCorrectness` is not set to `false`).
2. It includes the value of the text input as a response of the answer.


---

### Attribute Example: width


```doenet-example
<p>
  <textInput width="600px">
    <label>A wider input</label>
  </textInput>
</p>
<p>
  <textInput width="10px">
    <label>A narrower input</label>
  </textInput>
</p>
```



The default width of the input space can be customized with the `width` attribute.



---

### Attribute Example: height


```doenet-example
<p>
  <textInput height="600px" expanded>
    <label>A taller input</label>
  </textInput>
</p>
```



The default height of the input space can be customized with the `height` attribute. This must be used in combination with the `expanded` attribute.



---

### Attribute Example: labelIsName


```doenet-example
<p>Enter the shape that is shown.</p>

<graph size="small" displayXAxis="false" displayYAxis="false"> 
  <shortDescription>A triangle with a text input labeled "shape" for entering its name.</shortDescription>
  <textInput name="shape" labelIsName>
    <shortDescription>input for the name of the shape shown</shortDescription>
  </textInput>
  <regularPolygon numSides="3" sideLength="18" filled center="(-2,0)"/>
</graph>

<textInput name="Hi" labelIsName>
  <shortDescription>text input demonstrating labelIsName</shortDescription>
</textInput>
```



The `labelIsName` attribute can be used to assign an automatic label to a `<textInput/>{:dn}`.



---

### Attribute Example: draggable


```doenet-example
<p>Play with moving the boxes around.</p>

<graph size="small"> 
  <shortDescription>A circle with two text inputs, one fixed and one draggable.</shortDescription>
  <textInput 
    draggable="false" 
    prefill="cannot drag" 
    anchor="(-3,-5)"
  >
    <shortDescription>fixed text box</shortDescription>
  </textInput>
  <textInput prefill="drag me" anchor="(3,5)">
    <shortDescription>draggable text box</shortDescription>
  </textInput>
  <circle radius="8" filled/>
</graph>
```



The `draggable` attribute can be set to false to prevent users from moving the `<textInput/>{:dn}` on a `<graph>{:dn}`.



## Property Examples

### Property Example: value


```doenet-editor-horiz
<textInput name="ti">
  <shortDescription>any text</shortDescription>
</textInput>

<p><c>value</c> = $ti.value</p>
```



The `value` property contains the finalized value the user entered into the input.  It is updated when the user presses enter or navigates away.



---

### Property Example: immediateValue


```doenet-editor-horiz
<textInput name="ti">
  <shortDescription>any text</shortDescription>
</textInput>

<p><c>immediateValue</c> = $ti.immediateValue</p>
```



The `immediateValue` property contains the value the user enters into the input as they are typing.


---

### Property Example: attributes as properties


```doenet-editor-horiz
<graph size="small" displayXAxis="false" displayYAxis="false">
<shortDescription>A text input on a graph demonstrating attributes used as properties.</shortDescription>
<textInput 
  name="ti" 
  width="50" 
  prefill="type here"
  >
    <label>Name: </label>
</textInput>
</graph>

<p><c>prefill</c> = $ti.prefill</p>
<p><c>expanded</c> = $ti.expanded</p>
<p><c>width</c> = $ti.width</p>
<p><c>labelIsName</c> = $ti.labelIsName</p>
<p><c>draggable</c> = $ti.draggable</p>
<p><c>label</c> = $ti.label</p>
```



The attributes illustrated in the example are also accessible as properties. The value stored in a `<label>{:dn}` child can also be rendered using the `label` property.
