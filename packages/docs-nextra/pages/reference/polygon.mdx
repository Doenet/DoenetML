# `<polygon>{:dn}`

<ComponentDisplay name='polygon'/>

`<polygon>{:dn}` is a [Graphical](../document_structure/essentialConcepts#component-types)
component that renders a general polygon when nested inside a [`<graph>{:dn}`](graph.mdx) 
component. The `<polygon>{:dn}` tag is self-closing and the `vertices` attribute is required.
(See also, [`<regularPolygon>{:dn}`](regularPolygon.mdx).)

<AttrPropDisplay name='polygon'/>

## Examples

### Example: Default `<polygon>{:dn}` from vertices


```doenet-editor-horiz
<graph size="small">
  <shortDescription>A graph of a polygon with four vertices</shortDescription>
  <polygon vertices="(1,2) (3,5) (-2,6) (0,-5)"/>
</graph>
```



The default `<polygon>{:dn}` requires the `vertices` attribute as a minimum.



---

### Example: Repeat-generated `<polygon>{:dn}`


```doenet-editor-horiz
<p>
  <mathInput name="n"><label>How many vertices?</label></mathInput>
</p>

<graph size="small">
  <shortDescription>A graph of a polygon with a user-specified number of vertices</shortDescription>
  <repeatForSequence 
    name="points" 
    from="1" 
    to="$n">
      <point>
      (<sampleRandomNumbers 
         from="-8" to="8"/>, 
      <sampleRandomNumbers 
        from="-8" to="8"/>)
    </point>
  </repeatForSequence>

  <polygon vertices="$points"/>
</graph>
```



In this example, the inputs to the `vertices` attribute are generated by 
a `<repeatForSequence>{:dn}` within the `<graph>{:dn}`.



---

### Example: `<polygon>{:dn}` from added `<points>{:dn}`


```doenet-editor-horiz
<p>Drag the added vertices away from the origin to construct the polygon.</p>

<callAction actionName="addChildren" target="$g" anchor="(-8,-8)">
  <point>(0,0)</point>
  <label>Add vertex</label>
</callAction>

<callAction actionName="deleteChildren" target="$g" anchor="(7,-8)">
  <label>Delete vertex</label>
</callAction>

<graph name="g" size="small">
  <shortDescription>A graph of a polygon built from added vertices</shortDescription>
  <polygon vertices="$addedPts"/>
</graph>

<collect name="addedPts" from="$g" componentType="point"/>
```



In this example, the inputs to the `vertices` attribute are generated by a series of `<callAction>{:dn}` buttons that add a vertex at a time to the `<graph>{:dn}`.



## Attribute Examples

### Attribute Example: vertices


```doenet-example
<graph size="small">
  <shortDescription>A graph of a polygon</shortDescription>
  <polygon name="poly1" vertices="(1,2) (3,5) (-2,6) (0,-5)"/>
</graph>
<p><c>vertices</c> = <asList>$poly1.vertices</asList></p>
<p><c>vertices[2]</c> = $poly1.vertices[2]</p>
<p><c>vertices[2][2]</c> = $poly1.vertices[2][2]</p>
```



The `vertices` attribute takes a list of points as its input.



---

### Attribute Example: filled


```doenet-example
<graph size="small">
  <shortDescription>A graph of six filled polygons with different styles</shortDescription>
  <polygon name="p1" vertices="(-8,-4) (-8,5) (-6,2)" filled/>
  <polygon name="p2" vertices="(-5,-4) (-5,5) (-3,2)" filled styleNumber="2"/>
  <polygon name="p3" vertices="(-2,-4) (-2,5) (0,2)" filled styleNumber="3"/>
  <polygon name="p4" vertices="(1,-4) (1,5) (3,2)" filled styleNumber="4"/>
  <polygon name="p5" vertices="(4,-4) (4,5) (6,2)" filled styleNumber="5"/>
  <polygon name="p6" vertices="(7,-4) (7,5) (9,2)" filled styleNumber="6"/>
</graph>     

<p><c>filled</c> = $p1.filled</p>
```



The `filled` attribute renders the polygon with default shading to match the styleNumber.



---

### Attribute Example: draggable


```doenet-example
<p>Try dragging each of the polygons.</p>

<graph size="small">
  <shortDescription>A graph of a draggable polygon and a non-draggable polygon</shortDescription>
  <polygon name="poly1" vertices="(-6,2) (-3,-2) (3,5) (-1,4)"/>
  <polygon name="poly2" vertices="(6,-5) (4,7) (2,2)" styleNumber="2" draggable="false"/>
</graph>

<p>Blue polygon: <c>draggable</c> = $poly1.draggable</p>
<p>Red polygon: <c>draggable</c> = $poly2.draggable</p>      
```



The `draggable` attribute determines whether the `<polygon>{:dn}` can be dragged around the `<graph>{:dn}`. All graphical objects are draggable by default.



---

### Attribute Example: verticesDraggable


```doenet-example
<p>Try dragging each of the polygons and their vertices.</p>

<graph size="small">
  <shortDescription>A graph of two polygons illustrating verticesDraggable</shortDescription>
  <polygon name="poly1" vertices="(-6,2) (-3,-2) (3,5) (-1,4)"/>
  <polygon name="poly2" vertices="(6,-5) (4,7) (2,2)" styleNumber="2" verticesDraggable="false"/>
</graph>

<p>Blue polygon: <c>verticesDraggable</c> = $poly1.verticesDraggable</p>
<p>Red polygon: <c>vertices.Draggable</c> = $poly2.verticesDraggable</p>
```



Vertices are draggable by default. Use the `verticesDraggable` attribute set to false to 
disable users from changing the proportions of the `<polygon>{:dn}`. The polygon as a whole 
can still be repositioned on the `<graph>{:dn}`.



---

### Attribute Example: Standard graphical attributes


```doenet-editor-horiz
<graph size="small">
  <shortDescription>A graph illustrating standard graphical attributes on polygons</shortDescription>
  <circle styleNumber="5" filled radius="6" fixed/>
  <polygon 
    name="poly1" 
    vertices="(-6,2) (-3,-2) (3,5) (-1,4)" 
    hide="$hidePolygon1" 
    layer="1" 
    styleNumber="2" 
    applyStyleToLabel 
    labelIsName 
    fixLocation
  />
  <polygon 
    name="poly2" 
    vertices="(6,-5) (4,7) (2,2)" 
    styleNumber="3" 
    applyStyleToLabel 
    labelIsName
  />
</graph>

<booleanInput name="hidePolygon1"><label>Hide polygon1</label></booleanInput>
<p>Standard graphical attributes:</p>
    <ul>
      <li><attr>hide</attr></li>
      <li><attr>fixLocation</attr></li>
      <li><attr>layer</attr> (0) is default value.</li>
      <li><attr>styleNumber</attr> 1 - 6</li>
      <li><attr>applyStyleToLabel</attr></li>
      <li><attr>labelIsName</attr></li>
    </ul>
```



The uses of the following standard graphical attributes are illustrated: `hide`, `fixLocation`, `layer`, `styleNumber`, `applyStyleToLabel` and `labelIsName`.



## Property Examples

### Property Example: Style properties


```doenet-editor-horiz
<graph size="small">
  <shortDescription>A graph of a styled, filled polygon</shortDescription>
  <polygon 
    name="poly1" 
    vertices="(-6,2) (-3,-2) (3,5) (-1,4)" 
    styleNumber="3" 
    filled
  />
</graph>

<p><c>styleDescription</c> = $poly1.styleDescription</p>
<p><c>styleDescriptionWithNoun</c> = $poly1.styleDescriptionWithNoun</p>
<p><c>fillStyleDescription</c> = $poly1.fillStyleDescription</p>
<p><c>borderStyleDescription</c> = $poly1.borderStyleDescription</p>
```



The `styleDescription`, `styleDescriptionWithNoun`, `fillStyleDescription`, and `borderStyleDescription` properties render a text description corresponding to the `styleNumber` of the component.
