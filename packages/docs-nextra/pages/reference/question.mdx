# ` <question>{:dn}`

` <question>{:dn}` renders an auto-numbered block of content. Inline components such as paragraphs (` <p>{:dn}`) and other components can be nested within an ` <question>{:dn}` to group and order content. Some examples are: another ` <question>{:dn}`, a ` <section>{:dn}`, a ` <figure>{:dn}`, or an ` <example>{:dn}`.


The ` <question>{:dn}` component behaves identically to the ` <problem>{:dn}` component.

### Example: ` <question>{:dn}` with default title


```doenet-editor-horiz
<question>
  <p><answer>42</answer></p>
</question>
<section>
  <p><lorem generateWords="10"/></p>
</section>
<exercise>
  <p>Build a water rocket.</p>
</exercise>
```



Within a DoenetML document, default titles for sectional components are auto-numbered. Each sectional component is on the same counter.



---

### Example: ` <question>{:dn}` with custom title


```doenet-editor-horiz
<question><title>Factor the trinomial</title>
  <p><m>x^2 + 3x + 2 = </m>
    <answer symbolicEquality>
      <award>(x+2)(x+1)</award>
      <award>(x+1)(x+2)</award>
    </answer>
  </p>
</question>
```



A custom ` <question>{:dn}` title is creating by providing a ` <title>{:dn}` tag within the ` <question>{:dn}`.



---

### Example: Nesting sectional components


```doenet-editor-horiz
<question includeAutoNumber>
  <p>Move the point.</p>
  <graph size="tiny">
    <point name="p"/>
  </graph>
  <answer>
    <award><when>$p != (0,0)</when></award>
  </answer>
  <question includeAutoNumber includeParentNumber>
    <p>True or False? 
      <answer inline>
        <choice credit="1">T</choice>
        <choice>F</choice>
    </answer>
    </p>
  </question>
</question>
```



A hierarchy is created by nesting one or more sectional components within a ` <question>{:dn}`.

### Example: newNameSpace


```doenet-editor-horiz
<question boxed>
  <p>This is the same namespace as the larger document.</p>
  <number name="x">sqrt(2)</number>  
  <p><m>x = $x</m></p>
</question>
<question newNameSpace name="secondquestion" boxed>
  <p>This is a separate, distinct namespace.</p>
  <number name="x">3/4</number>  
  <p><m>x = $x</m></p>
</question>  
<p>General document: <m>x = $x</m></p>
<p>Second question: <m>x</m> = $(secondquestion/x)</p>
```



The `newNameSpace` attribute defines a separate namespace within the sectional component. Values corresponding to that namespace can be accessed outside the namespace by prepending the name of the question.



---

### Example: includeParentNumber


```doenet-editor-horiz
<section boxed>
  <question>
    <p>This question is nested inside a section. The default number does not include the parent number.</p>
  </question>
</section>
<section boxed>
  <question includeParentNumber="true">
    <p>This exercise is nested inside a section. Rendering of the parent section number has been included.</p>
  </question>
</section>
```



The `includeParentNumber` attribute is `false` by default for the ` <question>{:dn}` component (and `true` by default for the ` <section>{:dn}` component).


In order to include the number of the parent sectional component, set `includeParentNumber` to `true`.



---

### Example: includeAutoNumber


```doenet-editor-horiz
  <question boxed><title>Custom title</title>
    <p>In this question, the custom title overrides the default numbering of the question.</p>
  </question>
  <question boxed includeAutoNumber="true">
### Example: Special title

    <p>In this question, the number of the question is still rendered in addition to the custom title.</p>
  </question>
```



The `includeAutoNumber` attribute is `false` by default for the ` <question>{:dn}` component. This means that if a question contains a nested ` <title>{:dn}`, it will not render the default numbering (although it will still be counted in the numbering scheme behind the scenes).


In order to display the question number in addition to the custom ` <title>{:dn}`, set the `includeAutoNumber` attribute to `true`.

### Example: aggregateScores and sectionWideCheckWork


```doenet-editor-horiz
<question sectionWideCheckWork aggregateScores><title>Multiple questions</title>
  <p>Enter <m>x</m>: <answer>x</answer></p>
  <p>Enter <m>y</m>: <answer>y</answer></p>
  <p>Enter <m>z</m>: <answer>z</answer></p>
</question>
```



The `aggregateScores` and `sectionWideCheckWork` attributes together combine validation for all ` <answer>{:dn}` components within the ` <question>{:dn}` under a single "Check Work" button.

### Example: boxed


```doenet-editor-horiz
<question boxed><title>Boxed question</title>
<p>Move the point.</p>
  <graph size="tiny">
    <point name="p"/>
  </graph>
  <answer>
    <award><when>$p != (0,0)</when></award>
  </answer>  
</question>
```



The `boxed` attribute changes the default formatting for a sectional component to one that has a border and a shaded banner for the title.



---

### Example: hide


```doenet-editor-horiz
<booleanInput name="hidequestion">
  <label>Click here to hide the question</label>
</booleanInput>
<question hide="$hidequestion"><title>Now you see me...</title>
  <p>Type "sassafras" <answer type="text">sassafras</answer>.</p>
</question>
```



The `hide` attribute takes a boolean as input and can be used with any rendered component.
