# ` <conditionalContent>{:dn}`


` <conditionalContent>{:dn}` defines one or more blocks of content that 
are evaluated only when certain conditions are met. Conditions may be defined directly on the ` <conditionalContent>{:dn}` tag using the `condition` attribute.  Alternatively, one can use one or more [` <case>{:dn}`](case.mdx) children with an optional [` <else>{:dn}`](else.mdx) child.

### Example: ` <conditionalContent>{:dn}` with nested ` <case>{:dn}` children


```doenet-editor-horiz
  <p>Choose a letter: 
  <choiceInput inline name="letter">
    <choice>A</choice>
    <choice>B</choice>
    <choice>C</choice>
  </choiceInput>    
</p>
<conditionalContent>
  <case condition="$letter.selectedIndices = 1">
    <p>A is for apple.</p>
  </case>
  <case condition="$letter.selectedIndices = 2">
    <p>B is for balloon.</p>
  </case>
  <case condition="$letter.selectedIndices = 3">
    <p>C is for cat.</p>
  </case>  
</conditionalContent> 
```



The ` <conditionalContent>{:dn}` component defines a block of content containing individual ` <case>{:dn}` children. The doenetML within each ` <case>{:dn}` is only rendered when its specified `condition` is `true`.



---

### Example: assignNames


```doenet-editor-horiz
  <p>Type a number <m> > 0 </m>: <mathInput name="n"/></p>
  <conditionalContent condition="$n > 0" assignNames="x y" hide>
    <number>$n + 1</number>
    <number>$n - 1</number>
  </conditionalContent>
  <p><m>x = $x</m></p>
  <p><m>y = $y</m></p>
  <conditionalContent assignNames="z" hide>
    <case condition="$n > 0">
      <number>$n + 1</number>
    </case>
    <case condition="$n <= 0">
      <number>$n - 1</number>
    </case>
  </conditionalContent>
  <p><m>z = $z</m></p>
```



There are two applications of `assignNames` illustrated. In the first ` <conditionalContent>{:dn}`, if the specified `condition` is met, (i.e. if a positive number is entered) the ` <number>{:dn}` components defined within the ` <conditionalContent>{:dn}` are assigned the names `x` and `y` in sequence. If the `condition` is not met these ` <number>{:dn}` components are not defined.


In the second ` <conditionalContent>{:dn}` block, there are two different ` <case>{:dn}` components. The value $z$ is assigned depends on the content within the ` <case>{:dn}` whose `condition` is satisfied.

### Example: condition


```doenet-editor-horiz
<p>Let's graph a... 
    <choiceInput name="shape">
      <choice>circle</choice>
      <choice>square</choice>
      <choice>pentagon</choice>
    </choiceInput></p>
  <graph size="small">
    <conditionalContent condition="$shape.selectedValues = circle">
      <circle radius="4" filled/>
    </conditionalContent>
    <conditionalContent condition="$shape.selectedValues = square">
      <rectangle width="8" height="8" filled center="(0,0)" verticesDraggable="false"/>
    </conditionalContent>
    <conditionalContent condition="$shape.selectedValues = pentagon">
      <regularPolygon numVertices="5" sideLength="6" filled/>
    </conditionalContent>
  </graph>
```



A ` <conditionalContent>{:dn}` block with a ` <condition>{:dn}` attribute renders only when the `condition` is satisfied. This allows for use of a ` <conditionalContent>{:dn}` component without nested ` <case>{:dn}` components.
