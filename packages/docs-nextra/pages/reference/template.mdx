# ` <template>{:dn}`


The ` <template>{:dn}` component performs an operation on each of the [` <sources>{:dn}`](sources.mdx) 
within a [` <map>{:dn}`](map.mdx).  See also: [` <map>{:dn}`](map.mdx) documentation.


## Attributes and Properties 
 <AttrDisplay name='template'/> 
 <PropDisplay name='template'/>
 
---

### Example: A counting ` <template>{:dn}`


```doenet-editor-horiz
  <p>Count by <m>2</m>'s <mathInput name="N"/> times:</p>   
  <map>
    <template>
      <p><m>n_$i = <math simplify>2 $i</math></m></p>  
    </template>
    <sources indexAlias="i"><sequence from="1" to="$N"/></sources>
  </map>
```



The ` <template>{:dn}` defines the content to be repeated or mapped using values defined in the ` <sources>{:dn}` component. Here, a new paragraph containing an ` <m>{:dn}` and a ` <math>{:dn}` component is generated for each term in the sequence defined by the ` <sources>{:dn}` tag.



---

### Example: ` <template>{:dn}` with ` <map>{:dn}` and parallel behavior


```doenet-editor-horiz
<p>Where is Spot?</p>
  <map behavior="parallel">
    <template>
      <p>Is he $whereIsSpot?  No, a $whatIsThere is $whereIsSpot.</p>  
    </template>
    <sources alias="whereIsSpot">
      <text>behind the door</text>
      <text>inside the clock</text>
      <text>under the stairs</text>
    </sources>
    <sources alias="whatIsThere">
      <text>big brown bear</text>
      <text>green snake</text>
      <text>big lion</text>
    </sources>
  </map>
```



In this example, the ` <template>{:dn}` is used to generate a repeated ` <p>{:dn}` component (paragraph) with distinct ` <sources>{:dn}` within a ` <map>{:dn}`. The `behavior` attribute the ` <map>{:dn}` is set to `"parallel"` so that the two independent aliases are used simultaneously without combination.
