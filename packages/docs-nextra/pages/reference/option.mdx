# `<option>{:dn}`


`<option>{:dn}` is a [General Operator](../document_structure/essentialConcepts#component-types)
component that defines a selection element within a [`<select>{:dn}`](select.mdx) block. 
If only `<math>{:dn}` type components are to be selected, the `<option>{:dn}` 
component can be excluded and selection items can simply be listed with a space between list items. 
However, if multiple types or complex selections are required, the `<option>{:dn}` component is necessary.


When varying component types are used as options, such as 
a `<text>{:dn}`, `<function>{:dn}`, 
or `<number>{:dn}`, these tags are nested within the `<option>{:dn}` tag.

```doenet-editor-horiz
<select name="x">
  <option><math>a + b</math></option>
  <option><math>a - b</math></option>
  <option><math>a^2 b</math></option>
```

Three options for a variable name are nested within a `<select>{:dn}`. The 
type of each `<option>{:dn}` must be specified.

It is possible to group several components for selection within a single `<option>{:dn}` component.
In order to reference a specific component within a selected `<option>{:dn}`, use array notation. 
The first index of the array notation refers to the selected `<option>{:dn}` component, and the
second index refers to the specific component within the `<option>{:dn}` component.

```doenet-editor-horiz

<p>Selection n6 is $n[6]</p>
<p>Selection n7 is $n[7]</p>
```

The `<option>{:dn}` component is used with the `selectWeight` attribute. This attribute takes any numerical value and defaults to 1. In the case below, the `red` option is twice as likely to be selected during recursive selections (note that the `<select>{:dn}` component has attribute specification `withReplacement="true"`) as the `green` or `yellow` options.

### Attribute Example: selectForVariants


```doenet-editor-horiz
<variantControl numVariants="3" variantNames="apple banana cherry" />

<p>Selected option:
  <select name="s">
    <option selectForVariants="apple"><text>red</text></option>
    <option selectForVariants="banana"><text>yellow</text></option>
    <option selectForVariants="cherry"><text>dark red</text></option>
  </select>
</p>
```


`selectForVariants` pins this `<option>{:dn}` to specific named variants of
the document. Paired with `variantNames` on
[`<variantControl>{:dn}`](variantControl), it lets the author control
exactly which option appears in which variant — useful when the variants
need to line up with a specific story, image, or partner answer key
rather than being randomly assigned. In the example, the *apple* variant
always shows `red`, *banana* always shows `yellow`, and *cherry* always
shows `dark red`.

A few rules to keep in mind:

- It's all-or-nothing per `<select>{:dn}`: either no option uses
  `selectForVariants`, or every variant name declared by
  `<variantControl>{:dn}` is covered.
- Each variant name has to be listed in exactly as many options as the
  `<select>{:dn}`'s `numToSelect` (the default is 1). For
  `numToSelect="3"`, each variant name must appear in three options'
  `selectForVariants` lists.
- Multiple variant names are space-separated, and matching is
  case-insensitive.



---

### Attribute Example: selectForVariants with multiple choices per variant


```doenet-editor-horiz
<variantControl numVariants="3" variantNames="apple banana cherry" />

<p>Selected pair:
  <select name="s" numToSelect="2">
    <option selectForVariants="apple banana"><text>red</text></option>
    <option selectForVariants="apple cherry"><text>yellow</text></option>
    <option selectForVariants="banana cherry"><text>dark red</text></option>
  </select>
</p>
```


When `numToSelect` is greater than 1, each variant name has to appear in
exactly that many options' `selectForVariants` lists. Here each variant
name (`apple`, `banana`, `cherry`) appears in exactly two options, so the
*apple* variant produces `red, yellow`, *banana* produces `red, dark red`,
and *cherry* produces `yellow, dark red`.
