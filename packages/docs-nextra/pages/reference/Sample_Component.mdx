### Example: Render the absolute value of a math expression
To embed the example in an editor, use doenet-editor, which defaults to top/bottom.

```doenet-editor
<m>|5 - 8| = <abs>5 - 8</abs></m>
```

#### Brief description of the example:
The `<abs>` component is used to render the absolute value of a math expression.


<a id="mathInput-ex"></a>
### Example: Absolute value of `<mathInput>`
To embed the example in a viewer, use the DoenetViewer react component with doenetml embedded in the source attribute.
```doenet-viewer
    <p>Type any number: <mathInput name="userN"/></p>
    <p><m>|$userN| = <abs>$userN</abs></m></p>
```

#### Brief description of the example:
This component hides the source code and just renders the doenetML.

<a id="numProps-ex"></a>
### Example: Absolute value of numerical properties
This is what the DoenetExample component does...
```doenet-example
<graph>
    <point name="dp1">(4,4)</point>
    <line name="l1" through="$dp1 (0,0)"/>
</graph>
<p>The slope is <m>m = $l1.slope</m></p>
<p>The absolute value of the slope is <m>|m| = <abs>$l1.slope</abs></m></p>
```

Children of the `<abs>` component can include references to `<mathInput>` values or other `math` quantities, such as single-valued numerical properties of graphical objects. Invalid children are strings and tuples.  Test code [here](https://www.doenet.org/portfolioeditor/_J1J2N530LyIvVWx64xB8V/_AgzWwqGV6Yy9DfqMyGlFm).

<a id="function-ex"></a>
### Example: Absolute value of a function
```doenet-editor-horiz
<graph size="small" name="g2">
    <function name="f">(1/4)x^2 - 4</function>
    <function styleNumber="2">
        <abs>$f</abs>
    </function>
</graph>
```
The `<abs>` component is used to render the absolute value of a named `<function>` within a `<graph>`.

---
### Special usage notes

Note that the `<abs>` component cannot be used directly in attribute specification. You can, however, use the `|` notation within an attribute specification. 
You can also define a `<math>` component separately, which contains the `<abs>` component, and then reference it in an attribute specification.
