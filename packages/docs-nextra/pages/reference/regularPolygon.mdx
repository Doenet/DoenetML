# ` <regularPolygon>{:dn}`


` <regularPolygon>{:dn}` renders a regular polygon on a ` <graph>{:dn}`.

## Attributes and Properties 
 <AttrDisplay name='regularPolygon'/> 
 <PropDisplay name='regularPolygon'/>

---

### Example: Default ` <regularPolygon>{:dn}`


```doenet-editor-horiz
<p>Try dragging the vertices of the polygon</p>
<graph size="small" xmin="-2" xmax="2" ymin="-2" ymax="2">
  <regularPolygon name="defaultPolygon"/>
  <circle styleNumber="6" radius="$defaultPolygon.radius" center="$defaultPolygon.center"/>
</graph>
<p>$defaultPolygon.radius</p>
```



The default shape for a ` <regularPolygon>{:dn}` without any attributes specified is a triangle with radius 1 (where <em>radius</em> implies <em>circumradius</em> in this case), centered at the origin.



---

### Example: ` <regularPolygon>{:dn}` with user-specified sides


```doenet-editor-horiz
<p>How many sides? (Enter an integer): 
  <mathInput name="userN" prefill="3"/></p>
<p>What is the side length?: 
  <slider name="userL" initialValue="5"/></p>
<graph size="small">
  <regularPolygon name="userPolygon" numSides="$userN" sideLength="$userL"/>
</graph>
```



In this example, the number of sides and side length are specified by the user with a ` <mathInput/>{:dn}` and a ` <slider/>{:dn}` component, respectively. These values are then copied into the `numSides` and `sideLength` attributes of the ` <regularPolygon>{:dn}` using the `$name` copy macro. Note that the `prefill` attributes are required in this case; the values for these attributes cannot be null.

### Example: Construct a polygon on a given line segment


```doenet-editor-horiz
<graph size="small">
  <lineSegment name="segment" endpoints="(-3,-6) (3,-4)" layer="1"/>
  <regularPolygon name="userPolygon" numSides="6" vertices="$segment.endpoints[1] $segment.endpoints[2]" styleNumber="2"/>
  <point name="A" copySource="segment.endpoints[1]" styleNumber="2" labelIsName/>
  <point name="B" copySource="segment.endpoints[2]" styleNumber="2" labelIsName/>
</graph>
```



Two vertices only of a ` <regularPolygon>{:dn}` are specified in the `vertices` attribute by copying the endpoints of a defined ` <lineSegment>{:dn}`. Try tinkering with the number of sides in this example, or dragging the endpoints of the line segment.

### Example: center


```doenet-editor-horiz
<p>Try dragging the pentagon around the graph.</p>

  <regularPolygon name="pentagon" center="(3,3)" numSides="5" sideLength="4"/>
$pentagon.center{styleNumber="2"}
</graph>     
```



The center of a regular polygon can be specified with the `center` attribute.


Note on precedence: the `vertices` and `center` attributes take precedence over the following attributes (in the event of overconstraint): `radius / circumRadius`, `inRadius / apothem`, `sideLength`, `perimeter`, and `area`.

The endpoints of a known side can be specified with the `vertices` attribute, and Doenet will construct the resulting regular polygon (given a specified number of sides).


Note on precedence: the `vertices` and `center` attributes take precedence over the following attributes (in the event of overconstraint): `radius / circumRadius`, `inRadius / apothem`, `sideLength`, `perimeter`, and `area`.

### Example: radius / circumRadius


```doenet-editor-horiz
<graph size="small">
  <regularPolygon name="square1" radius="4" numSides="4"  center="(-5,0)"/>
  <circle name="circ1" center="$square1.center" radius="$square1.radius"/>
  <regularPolygon name="square2" radius="4" numSides="4"  center="(5,0)" styleNumber="2"/>
  <circle name="circ2" center="$square2.center" radius="$square2.radius" styleNumber="2"/>
</graph> 
<p><c>radius 1</c> = $square1.radius</p>
<p><c>circumRadius 1</c> = $square1.circumRadius</p>
<p><c>radius 2</c> = $square2.radius</p>
<p><c>circumRadius 2</c> = $square2.circumRadius</p>      
```



The `radius` and `circumRadius` attributes are an interchangeable way to specify the distance from the center of a regular polygon to any of its vertices. (Note that the `vertices` attribute takes precedence in the case of overconstraint.)

Vertices are draggable by default. Use the `verticesDraggable` attribute set to false to disable users from changing the size and rotation of the ` <regularPolygon>{:dn}`. The polygon as a whole can still be repositioned on the ` <graph>{:dn}`.

### Example: fixed / fixLocation


```doenet-editor-horiz
<graph size="small" displayXAxis="false" displayYAxis="false">
  <regularPolygon name="fixLocationTri" fixLocation center="(-5,0)" sideLength="5" styleNumber="$styles" filled>
    <label>A</label>
  </regularPolygon>
  <regularPolygon name="fullyFixedTri" fixed center="(5,0)" sideLength="5" styleNumber="$styles" filled>
    <label>B</label>
  </regularPolygon>
</graph> 
<updateValue target="fixLocationTri.hide" newValue="true" type="boolean" disabled="$fixLocationTri.hide = true">
  <label>Hide polygon A</label>
</updateValue>
<updateValue target="fixLocationTri.hide" newValue="false" type="boolean" disabled="$fixLocationTri.hide = false">
  <label>Show polygon A</label>
</updateValue>
<updateValue target="fullyFixedTri.hide" newValue="true" type="boolean">
  <label>Hide polygon B</label>
</updateValue>
<p><c>fixed (A)</c> = $fullyFixedTri.fixed</p>
<p><c>fixLocation (B)</c> = $fixLocationTri.fixLocation</p>    
```



This example illustrates the difference between the `fixed` and the `fixLocation` attributes. If an object is `fixed` it cannot be moved or changed in any way. The `styleNumber`, `hide`, and `filled` attributes, for example, cannot be changed or updated. This is why the ` <updateValue>{:dn}` button for polygon B is ineffective. 


With `fixLocation`, by contrast, the object cannot be moved or resized (i.e. it automatically makes vertices not draggable), but other characteristics of the object can still be changed.

### Example: Standard graphical attributes


```doenet-editor-horiz
<graph size="small">
  <circle styleNumber="5" filled radius="6" fixed/>
  <regularPolygon name="A" layer="1" styleNumber="3" applyStyleToLabel labelIsName hide="$hidePolygonA" draggable="false" filled sideLength="4" numSides="8" center="(-3,-4)"/>
  <regularPolygon name="B" applyStyleToLabel styleNumber="2" labelIsName sideLength="2" numSides="8" center="(6,4)"/>
</graph>
<booleanInput name="hidePolygonA" labelIsName/>
<p>Standard graphical attributes:</p>
    <ul>
      <li><attr>hide</attr></li>
      <li><attr>draggable</attr></li>
      <li><attr>layer</attr> (0) is default value.</li>
      <li><attr>styleNumber</attr> 1 - 6</li>
      <li><attr>applyStyleToLabel</attr></li>
      <li><attr>labelIsName</attr></li>
    </ul>
```



The uses of the following standard graphical attributes are illustrated: `hide`, `draggable`, `layer`, `styleNumber`, `applyStyleToLabel` and `labelIsName`.


Layers: By adjusting the `layer` attribute number, geometry can be placed in front of or behind other geometry. The default `layer` number is 0; all higher numbers are placed in front. Due to the fact that the default styleNumbers all involve a degree of transparency, this distinction may not be readily visible.

### Example: vertices


```doenet-editor-horiz
<graph size="small">
  <regularPolygon name="largePoly" numSides="13" vertices="(-4,-2) (-2,-4)"/>
  $largePoly.vertices{styleNumber="5"}
</graph>
<p><c>vertices</c> = <asList>$largePoly.vertices</asList></p>
<p><c>vertices[3]</c> = $largePoly.vertices[3]</p>
<p><c>vertices[3].y</c> = $largePoly.vertices[3].y</p>
```



The `vertices` property stores/renders all the computed vertices of the regular polygon in an array. Array notation can be used to access an individual vertex or a coordinate of a vertex.
