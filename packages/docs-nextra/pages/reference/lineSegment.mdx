# `<lineSegment>{:dn}`

<ComponentDisplay name='lineSegment'/>

`<lineSegment>{:dn}` is a [Graphical](../concepts/essentialConcepts#component-types)
component that renders a line segment when nested within a [`<graph>{:dn}`](graph.mdx) component.

<AttrPropDisplay name='lineSegment'/>

## Examples

### Example: Default `<lineSegment>{:dn}`


```doenet-example
<graph size="small" xmin="-2" xmax="2" ymin="-2" ymax="2">
  <shortDescription>graph with a default line segment</shortDescription>
  <lineSegment/>
</graph>
```



Without any attributes, the default `<lineSegment>{:dn}` has endpoints $(0,0)$ and $(1,0)$.



---

### Example: `<lineSegment>{:dn}` with rendered endpoints


```doenet-example
<graph size="small">
  <shortDescription>graph of a line segment with its endpoints marked</shortDescription>
  <lineSegment name="segment1" endpoints="(0,0) (6,6)"/>
  <pointList styleNumber="2">
    $segment1.endpoints
  </pointList>
</graph>
```



The `endpoints` property of a named `<lineSegment>{:dn}` can be 
rendered on a `<graph>{:dn}` with the reference `$name.property`.



---

### Example: `<lineSegment>{:dn}` constrained to a grid


```doenet-editor-horiz
<graph size="small" grid="1 1">
  <shortDescription>graph of a line segment whose endpoints are constrained to a grid</shortDescription>
  <point name="A" styleNumber="4">(0,0)
    <constrainToGrid/>
  </point>
  <point name="B" styleNumber="4">(6,6)
    <constrainToGrid/>
  </point>
  <lineSegment endpoints="$A $B"/>
</graph>
```



The endpoints of a `<lineSegment>{:dn}` are predefined as constrained points, 
thereby constraining the segment to a grid as well.



## Attribute Examples

### Attribute Example: endpoints


```doenet-editor-horiz
<graph size="small">
  <shortDescription>graph of a line segment with specified endpoints</shortDescription>
  <lineSegment name="segment1" endpoints="(1,2) (3,4)"/>
</graph>

<p><c>endpoints</c> = $segment1.endpoints</p>
<p><c>endpoints[2]</c> = $segment1.endpoints[2]</p>
<p><c>endpoints[2][2]</c> = $segment1.endpoints[2][2]</p>
```



The `endpoints` attribute takes as an input a list of two points.



---

### Attribute Example: draggable


```doenet-example
<p>Try dragging each of the line segments.</p>
<graph size="small">
  <shortDescription>graph with a draggable and a non-draggable line segment</shortDescription>
  <lineSegment name="segment1" endpoints="(0,0) (6,6)"/>
  <lineSegment name="segment2" draggable="false" endpoints="(-2,4) (5,-2)" styleNumber="2"/>
</graph>

<p>Blue lineSegment, <c>draggable</c> = $segment1.draggable</p>
<p>Red lineSegment, <c>draggable</c> = $segment2.draggable</p>
```



The `draggable` attribute determines whether the `<lineSegment>{:dn}` can be 
dragged around the `<graph>{:dn}`. All graphical objects are draggable by default.



---

### Attribute Example: endpointsDraggable


```doenet-example
<p>Try dragging the endpoints of each of the line segments.</p>
<graph size="small">
  <shortDescription>graph with one line segment whose endpoints are draggable and one whose endpoints are not</shortDescription>
  <lineSegment name="segment1" endpoints="(0,0) (6,6)"/>
  <lineSegment name="segment2" endpointsDraggable="false" endpoints="(-2,4) (5,-2)" styleNumber="2"/>
</graph>

<p>Blue lineSegment, <c>endpointsDraggable</c> = $segment1.endpointsDraggable</p>
<p>Red lineSegment, <c>endpointsDraggable</c> = $segment2.endpointsDraggable</p>
```



Endpoints are draggable by default. Use the `endpointsDraggable` attribute set to 
false to disable users from changing the shape and length of the `<lineSegment>{:dn}`. The segment 
as a whole can still be repositioned on the `<graph>{:dn}`.



---

### Attribute Example: showCoordsWhenDragging


```doenet-example
<p>Drag the endpoints of each of the line segments.</p>
<graph size="small">
  <shortDescription>graph with two line segments demonstrating the showCoordsWhenDragging behavior</shortDescription>
  <lineSegment name="segment1" endpoints="(0,0) (6,6)"/>
  <lineSegment name="segment2"  endpoints="(-2,4) (5,-2)" styleNumber="2" showCoordsWhenDragging="false"/>
</graph>

<p>Blue lineSegment, <c>showCoordsWhenDragging</c> = $segment1.showCoordsWhenDragging</p>
<p>Red lineSegment, <c>showCoordsWhenDragging</c> = $segment2.showCoordsWhenDragging</p>
```



By default, a light-grey set of coordinates is shown while the user drags the 
endpoints of the line segment. To turn off this default behavior, set the `showCoordsWhenDragging` attribute 
to false.



---

### Attribute Example: slope and length


```doenet-editor-horiz
<graph size="small" xmin="-1" xmax="5" ymin="-1" ymax="5">
  <shortDescription>graph of a line segment defined by a slope and a length</shortDescription>
  <lineSegment name="segment1" slope="1" length="4"/>
</graph>

<p><c>slope</c> = $segment1.slope</p>
<p><c>length</c> = $segment1.length</p>
```



When no `endpoints` are specified, the `slope` and `length` attributes define the 
segment: the first endpoint is placed at the origin and the second is placed a distance 
`length` away in the direction determined by `slope`. A negative `length` points the 
segment in the opposite direction.



---

### Attribute Example: midpoint


```doenet-editor-horiz
<graph size="small" xmin="-1" xmax="5" ymin="-1" ymax="5">
  <shortDescription>graph of a line segment defined by one endpoint and its midpoint</shortDescription>
  <lineSegment name="segment1" endpoints="(1,2)" midpoint="(2,3)"/>
  <point styleNumber="2">$segment1.center</point>
</graph>

<p><c>endpoints</c> = $segment1.endpoints</p>
<p><c>center</c> = $segment1.center</p>
```



The `midpoint` attribute specifies a point that lies on the segment, which is its 
midpoint by default. Combined with a single `endpoints` value, the second endpoint is 
placed so that the given point is the midpoint of the segment: here the endpoint 
$(1,2)$ and midpoint $(2,3)$ produce a second endpoint at $(3,4)$. The `midpoint` can 
also be paired with `slope` and `length` (with no `endpoints`) to center a segment of a 
given direction and length on a point.



---

### Attribute Example: midpointOffset


```doenet-editor-horiz
<graph size="small" xmin="-5" xmax="5" ymin="-3" ymax="3">
  <shortDescription>graph showing how midpointOffset slides the reference point along a line segment</shortDescription>
  <point name="P" styleNumber="2">(0,0)</point>
  <lineSegment name="segment1" midpoint="$P" slope="0" length="4" midpointOffset="$offset"/>
</graph>

<p>Set <c>midpointOffset</c> (clamped to the range $[-1,1]$): <mathInput name="offset" prefill="0"/></p>
<p><c>endpoints</c> = $segment1.endpoints</p>
```



The `midpointOffset` attribute controls where the `midpoint` point sits along the 
segment: `0` (the default) is the midpoint, `-1` is the first endpoint, and `1` is the 
second endpoint. With the point $P=(0,0)$ held fixed and a horizontal segment of length 
$4$, an offset of $0$ gives endpoints $(-2,0)$ and $(2,0)$; an offset of $-1$ gives 
$(0,0)$ and $(4,0)$; and an offset of $1$ gives $(-4,0)$ and $(0,0)$.
