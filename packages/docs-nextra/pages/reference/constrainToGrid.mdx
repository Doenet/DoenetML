# ` <constrainToGrid/>{:dn}`

` <constrainToGrid>{:dn}` is used to constrain a [` <point>{:dn}`](point.mdx) to a grid. The grid can be the default $1 \times 1$ grid, or a custom grid defined by the `dx` and `dy` attributes, or by the `xOffset` and `yOffset` attributes.

` <constrainToGrid/>{:dn}` forces points to snap to gridline locations. The ` <constrainToGrid/>{:dn}` component is nested within a [` <constraints>{:dn}`](constraints.mdx) block.

### Example: Constrain a ` <point>{:dn}` to the default grid


```doenet-editor-horiz
<p>Move the point.</p>
<graph size="small">
  <point name="p">
    <constraints>
      <constrainToGrid/>
    </constraints>
    <label>$p.coords</label>
  </point>
</graph>
```



The ` <point>{:dn}` is constrained to the default $1 \times 1$ grid.



---

### Example: Constrain a ` <circle>{:dn}` to the default grid


```doenet-editor-horiz
<p>Move the circle.</p>
<graph size="small">
  <point name="p" hide>
    <constraints>
      <constrainToGrid/>
    </constraints>
  </point>
  <circle name="circ" center="$p" radius="3" filled/>
</graph>
<p>The center of the circle is $circ.center.</p>
```



To constrain geometric objects other than points, first define a named ` <point>{:dn}` with the desired constraint, and then reference that point within the definition of the geometric object. The original ` <point>{:dn}` can be hidden using the `hide` attribute.



---

### Example: Constrain a ` <point>{:dn}` to a custom grid


```doenet-editor-horiz
<p>Move the point.</p>
<graph size="small" grid="1 1" 
  xmin="-3" xmax="3" ymin="-3" ymax="3">
  <point name="p">
    <constraints>
      <constrainToGrid dx="0.5" dy="0.5"/>
    </constraints>
    <label>$p.coords</label>
  </point>
</graph>
```



The ` <point>{:dn}` is constrained to a custom $0.5 \times 0.5$ grid.



---

### Example: dx, dy


```doenet-editor-horiz
<p>Move the point.</p>
<graph size="small" grid="0.5 2" xmin="-3" xmax="3" 
  ymin="-1" ymax="5">
  <point name="p">
    <constraints>
      <constrainToGrid dx="0.5" dy="2"/>
    </constraints>
    <label>$p.coords</label>
  </point>
</graph>
```



The ` <point>{:dn}` is constrained to a custom grid using the `dx` and `dy` components. The `grid` specified for the ` <graph>{:dn}` does not affect the constraint grid, it is merely adjusted to match in this example for illustration.



---

### Example: xOffset, yOffset


```doenet-editor-horiz
<p>Move the point.</p>
<graph size="small" grid="1 1" xmin="-3" xmax="3" 
  ymin="-3" ymax="3">
  <point name="p">
    <constraints>
      <constrainToGrid xOffset="0.5" yOffset="0.5"/>
    </constraints>
    <label>$p.coords</label>
  </point>
</graph>
```



The ` <point>{:dn}` is constrained to the default $1 \times 1$ grid with a specified `xOffset` and `yOffset` of $0.5$.
