# ` <constraints>{:dn}`

` <constraints>{:dn}` is a [Graphical](../document_structure/essentialConcepts#component-types) 
component that delineates a block of code for defining one or more constraint components 
associated with a ` <point>{:dn}`.

## Attributes and Properties 
The ` <constraints>{:dn}` component has no attributes.
 <PropDisplay name='constraints'/>

---

### Example: ` <constraints>{:dn}` with nested `<constrainToGrid/>{:dn}`


```doenet-editor-horiz
<p>Move the point.</p>
<graph size="small">
  <point name="p">
    <constraints>
      <constrainToGrid/>
    </constraints>
    <label>$p.coords</label>
  </point>
</graph>
```



The ` <point>{:dn}` is constrained to the 
default $1 \times 1$ grid by nesting a ` <constrainToGrid/>{:dn}` 
component within a ` <constraints>{:dn}` block.



---

### Example: ` <constraints>{:dn}` with nested ` <constrainTo>{:dn}`


```doenet-editor-horiz
<p>Drag the $circ.styleDescriptionWithNoun along the line.</p>
<graph size="small">
  <line name="line1">y = x</line>
  <point name="p" hide>
    <constraints>
      <constrainTo>$line1</constrainTo>
    </constraints>
  </point>
  <circle name="circ" center="$p" radius="3" styleNumber="2" filled/>
</graph>
```



A ` <point>{:dn}` defining the center of the ` <circle>{:dn}` is 
constrained to a ` <line>{:dn}` by nesting a ` <constrainTo>{:dn}` component 
within a ` <constraints>{:dn}` block.



---

### Example: ` <constraints>{:dn}` with nested `<attractToGrid/>{:dn}`


```doenet-editor-horiz
    <graph grid="2 4">
      <point>
        <constraints>
          <attractToGrid dx="2" dy="4"/>
        </constraints>
      </point>
    </graph>
```



The ` <point>{:dn}` is attracted to a custom grid by 
nesting an ` <attractToGrid>{:dn}` component within a ` <constraints>{:dn}` block.



---

### Property Example: constraintUsed


```doenet-editor-horiz
<p>Move the point on and off the gridlines to 
observe the change in the <tag>constraints</tag> property 
(shown below the graph).
</p>

<graph size="small" grid="3 4">
  <point name="p">
    <constraints name="pConstraints">
      <attractToGrid dx="3" dy="4"/>
    </constraints>
    <label>$p.coords</label>
  </point>
</graph>
<p><c>constraintUsed</c> = 
  $pConstraints.constraintUsed</p>
```



The `constraintUsed` property renders the boolean value 
corresponding to whether the current location of the ` <point>{:dn}` is 
utilizing the specified constraint - in this case, 
an ` <attractToGrid/>{:dn}` constraint.
