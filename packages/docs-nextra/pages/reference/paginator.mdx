# ` <paginator>{:dn}`

` <paginator>{:dn}` is a [Sectional](../document_structure/essentialConcepts#component-types)
component that organizes content into discrete pages, allowing users to navigate through them one at a time.
To create navigation buttons for the paginator, this component is typically used in conjunction with the 
`<paginatorControls>{:dn}` component.

## Attributes and Properties

 <PropDisplay name='paginator'/>

---

### Example: A basic ` <paginator>{:dn}` with bottom navigation


```doenet-editor-horiz
<paginator name="pages">
  <section boxed>
    <lorem generateParagraphs="2"/>
  </section>
  <problem boxed>
    <lorem generateParagraphs="2"/>
  </problem>
  <exercise boxed>
    <lorem generateParagraphs="2"/>
  </exercise>
</paginator>

<paginatorControls paginator="$pages"/>
```

A `<paginator>{:dn}` component creates a separate page for each sectional component it contains. 
Sectional components are container elements for content, such as a `<section>`, `<problem>`, or `<exercise>`.
Users navigate through the pages using buttons generated by the `<paginatorControls>{:dn}` component.

The `<paginator>{:dn}` component must be assigned a name, which is then referenced
by the `paginator` attribute of the `<paginatorControls>{:dn}` component.

### Example: A ` <paginator>{:dn}` that advances based on user interactions


```doenet-example
<callAction actionName="setPage" 
  target="$steps" 
  number="2" 
  triggerWhen="$P.coords != (1,2)"/>

<callAction actionName="setPage" 
  target="$steps" 
  number="3" 
  triggerWhen="$ans1.creditAchieved"/>

<paginator name="steps">
  <section boxed><title>Move the point to see the next step</title>
    <graph>
      <point name="P">(1,2)
        <label>$P.coords</label>
      </point>
    </graph>
  </section>
  <section boxed><title>Enter <m>x</m> to see the next step</title>
    <p>Enter x: <answer name="ans1">x</answer></p>
  </section>
  <section boxed>
    <p>Nice job! You can now go back and look at your responses.</p>
  </section>
</paginator>
<paginatorControls paginator="$steps" disabled="$ans1.creditAchieved != 1"/>


```

The `setPage` actionName for a `<callAction>` component can be used with a `<paginator>{:dn}` to 
programmatically change the current page based on user interactions. Within the `<callAction>`, use the `number` 
attribute to specify the page number to navigate to, and the `triggerWhen` attribute to 
define the condition that triggers the page change.

### Attribute Example: initialPage


```doenet-editor-horiz
<paginator name="pages" initialPage="2">
  <section>
    <title>Graph of <m>y = $f1</m></title>
  <graph>
    <function name="f1">-x^2 + 2</function>
    <point name="P">$f1.maxima
      <label><m>$P.coords</m></label>
    </point>
  </graph>
  </section>
  <problem boxed>
    <p>What is the vertex of 
      <m>y = $f1</m> ?</p>
    <p>Vertex = 
      <answer>$f1.maxima</answer>
    </p>
  </problem>
</paginator>
<paginatorControls paginator="$pages"/>
```



The `initialPage` attribute specifies the page number value that the user will see when the 
`<paginator>{:dn}` is first rendered. 

---

### Property Example: currentPage, numPages


```doenet-editor-horiz
<paginator name="pages">
  <section boxed>
    <lorem generateParagraphs="2"/>
  </section>
  <problem boxed>
    <lorem generateParagraphs="2"/>
  </problem>
  <exercise boxed>
    <lorem generateParagraphs="2"/>
  </exercise>
</paginator>
<paginatorControls paginator="$pages"/>

<p><c>currentPage</c> = $pages.currentPage</p>
<p><c>numPages</c> = $pages.numPages</p>
```



The `currentPage` property returns the current page number being displayed by the ` <paginator>{:dn}` component, 
while the  `numPages` property returns the number of total pages in the ` <paginator>{:dn}`.
