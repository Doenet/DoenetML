# `<point>{:dn}`

<ComponentDisplay name='point'/>

`<point>{:dn}` is a [Graphical](../concepts/essentialConcepts#component-types)
component that renders a point on a `<graph>{:dn}`.


<AttrPropDisplay name='point'/>

## Examples

### Example: Default `<point>{:dn}`


```doenet-editor-horiz
<p>Drag the point to change its location.</p>
<graph size="small">
  <shortDescription>A graph of a draggable point</shortDescription>
  <point name="p"/>
</graph>

<p><m>P = $p</m></p>
```



Here a default `<point>{:dn}` is illustrated without any additional attributes.



---

### Example: `<point>{:dn}` with nested coordinates


```doenet-editor-horiz

  <point name="p">(2,3)</point>
</graph>

<p><m>P = $p</m></p>
```



The initial coordinates of a `<point>{:dn}` may be specified between opening and closing tags.


Point labels can be created either with the shortcut attribute `labelIsName` or by

### Example: `<point>{:dn}` constrained to a grid


```doenet-editor-horiz
<p><m>A = $A</m> (Constrained to grid)</p>
<p><m>B = $B</m> (Attracted to grid)</p>
<graph size="small" grid="1 1" xmin="-5" xmax="5" ymin="-5" ymax="5">
  <shortDescription>A graph of two points, one constrained to a grid and one attracted</shortDescription>
  <point name="A" labelIsName>(-2,2)
    <constrainToGrid/>
  </point>
  <point name="B" labelIsName>(2,2)
    <attractToGrid/>
  </point>
</graph>
```



Points can take constraints as children. Several different types of constraints are possible; in this example, the `<constrainToGrid/>{:dn}` and `<attractToGrid/>{:dn}` constraints are illustrated.

### Example: `<point>{:dn}` constrained to a `<line>{:dn}`


```doenet-editor-horiz

  </point>

```



Points can take constraints as children. Several different types of constraints are possible. In this case, the `<constrainTo>{:dn}` constraint references a named `<line>{:dn}`.

### Example: An animated `<point>{:dn}`


```doenet-editor-horiz
<p><m>p = $p</m></p>
<p><number name="t">-4pi</number></p>
<graph size="small" xmin="-5pi" xmax="5pi" ymin="-2.5" ymax="2.5" xTickScaleFactor="pi">
  <shortDescription>A graph of an animated point on a sine curve</shortDescription>
  <point name="p" styleNumber="2">
    ($t, sin($t))
  </point>
  <booleanInput name="on" anchor="(-3pi, 2)">
    <label>Animate point</label>
  </booleanInput>
</graph>
<animateFromSequence target="$t" from="-4pi" to="4pi" step="pi/16" animationInterval="100" animationOn="$on"/>
```



A `<point>{:dn}` is animated by paraterizing its coordinates with respect to the `<number>{:dn}` `t`. The parameter is animated using the `<animateFromSequence/>{:dn}` component.



---

### Example: Points that break the universe


```doenet-editor-horiz
<graph>
  <shortDescription>A graph of three interdependent points A, B, and C</shortDescription>
  <point name="A" labelIsName>($C.y, 7)</point>
  <point name="B" labelIsName>($A.y, -2)</point>
  <point name="C" labelIsName>($B.y, 5)</point>
</graph>
<p><mathInput bindValueTo="$A.x"><label><m>A_x = </m></label></mathInput>,
  <mathInput bindValueTo="$A.y"><label><m>A_y = </m></label></mathInput></p>
<p><mathInput bindValueTo="$B.x"><label><m>B_x = </m></label></mathInput>,
  <mathInput bindValueTo="$B.y"><label><m>B_y = </m></label></mathInput></p>
<p><mathInput bindValueTo="$C.x"><label><m>C_x = </m></label></mathInput>,
  <mathInput bindValueTo="$C.y"><label><m>C_y = </m></label></mathInput></p>
```



Although point C depends on point B which depends on point A which, in turn, depends on point C, Doenet can determine that there is actually no circular dependency in their relationship.


This example also illustrates how one can change a point either by dragging it or by changing a `<mathInput/>{:dn}` that is bound to one of its coordinates.



## Attribute Examples

### Attribute Example: draggable


```doenet-example
<graph size="small">
  <shortDescription>A graph of a draggable point and a non-draggable point</shortDescription>
  <point>(-6,0)
    <label>I'm free!</label>
  </point>
  <point draggable="false" styleNumber="2">
    (6,0)
    <label>I'm stuck!</label>
  </point>
</graph>
```



The `draggable` attribute determines whether the `<point>{:dn}` can be dragged around the `<graph>{:dn}`. All graphical objects are draggable by default.



---

### Attribute Example: x, y


```doenet-example
<p><m>p = $p</m></p>

<graph size="small">
  <shortDescription>A graph of a point at (3,2)</shortDescription>
  <point name="p" x="3" y="2"/>
</graph>
```



The `x` and `y` attributes are used to specify isolated `<point>{:dn}` coordinates.



---

### Attribute Example: coords


```doenet-example
<p><m>p = $p</m></p>

<graph size="small">
  <shortDescription>A graph of a point at (2,3)</shortDescription>
  <point name="p" coords="(2,3)"/>
</graph>
```



The `coords` attribute specifies the `<point>{:dn}` coordinates.



---

### Attribute Example: xs


```doenet-example
<p><m>p = $p</m></p>

<graph size="small">
  <shortDescription>A graph of a point with xs="2 3"</shortDescription>
  <point name="p" xs="2 3"/>
</graph>
```



The `xs` attribute specifies the point's location as a list of displacements.



---

### Attribute Example: labelPosition


```doenet-example
<graph size="small" displayYAxis="false" displayXAxis="false">
  <shortDescription>A graph of eight labeled points showing label positions</shortDescription>
  <point name="A" labelPosition="upperleft" coords="(-4,4)" labelIsName/>
  <point name="B" labelPosition="upperright" coords="(4,4)" labelIsName/>
  <point name="C" labelPosition="top" coords="(0,4)" labelIsName/>
  <point name="D" labelPosition="left" coords="(-4,0)" labelIsName/>
  <point name="E" labelPosition="right" coords="(4,0)" labelIsName/>
  <point name="F" labelPosition="lowerleft" coords="(-4,-4)" labelIsName/>
  <point name="G" labelPosition="bottom" coords="(0,-4)" labelIsName/> 
  <point name="H" labelPosition="lowerright" coords="(4,-4)" labelIsName/>
</graph>
```



The `labelPosition` attribute specifies the location of the point's label. Valid values are `upperleft`, `upperright`, `top`, `left`, `right`, `lowerleft`, `bottom`, `lowerright`. The default value is `upperright`.



---

### Attribute Example: showCoordsWhenDragging


```doenet-example
<p>Drag each of the points to observe the difference.</p>
<graph size="small">
  <shortDescription>A graph of two points illustrating showCoordsWhenDragging</shortDescription>
  <point name="A" coords="(-6,4)">
    <label>Showing</label>
  </point>
  <point name="B" showCoordsWhenDragging="false" coords="(4,4)">
    <label>Not showing</label>
  </point>
</graph>
```



By default, a light-grey set of coordinates is shown while the user drags the point. To turn off this default behavior, set the `showCoordsWhenDragging` attribute to false.



---

### Attribute Example: hideOffGraphIndicator


```doenet-example
<p>To see both points A and B, use the (-) button at the bottom of the graph to zoom out.</p>
<graph size="small">
  <shortDescription>A graph of two off-graph points with different indicator settings</shortDescription>
  <point name="A" labelIsName coords="(-15,4)" hideOffGraphIndicator/>
  <point name="B" labelIsName coords="(15,4)"/>
</graph>
```



By default, when a `<point>{:dn}` is dragged outside the extents of the `<graph>{:dn}`, or when a point is located outside the extents, an triangle-shaped indicator appears at the edge of the `<graph>{:dn}`. The user can either click this indicator to bring the `<point>{:dn}` back to the position of the cursor, or zoom out until the `<point>{:dn}` is visible again. 


In this example, only one indicator appears initially, since point A has the `hideOffGraphIndicator` attribute specified. The student would not be aware that point A exists, until zooming out sufficiently far on the graph.



---

### Attribute Example: markerStyle


```doenet-example
<graph size="small" displayXAxis="false" displayYAxis="false">
  <shortDescription>A graph of points drawn with different marker shapes</shortDescription>
  <point coords="(-6,1)" markerStyle="circle"><label>circle</label></point>
  <point coords="(-2,3)" markerStyle="square"><label>square</label></point>
  <point coords="(2,5)" markerStyle="triangle"><label>triangle</label></point>
  <point coords="(6,7)" markerStyle="diamond"><label>diamond</label></point>
  <point coords="(-6,-7)" markerStyle="triangleDown"><label>triangleDown</label></point>
  <point coords="(-2,-5)" markerStyle="triangleLeft"><label>triangleLeft</label></point>
  <point coords="(2,-3)" markerStyle="cross"><label>cross</label></point>
  <point coords="(6,-1)" markerStyle="plus"><label>plus</label></point>
</graph>
```



The `markerStyle` attribute sets the shape of the marker drawn at the point. Valid values are `circle`, `square`, `triangle` (an alias for `triangleUp`), `triangleUp`, `triangleDown`, `triangleLeft`, `triangleRight`, `diamond`, `cross`, and `plus`. The default value is `circle`.



---

### Attribute Example: markerSize


```doenet-example
<graph size="small" displayXAxis="false" displayYAxis="false">
  <shortDescription>A graph of points drawn at increasing marker sizes</shortDescription>
  <point coords="(-6,0)" markerSize="3"><label>3</label></point>
  <point coords="(-2,0)" markerSize="6"><label>6</label></point>
  <point coords="(2,0)" markerSize="10"><label>10</label></point>
  <point coords="(6,0)" markerSize="16"><label>16</label></point>
</graph>
```



The `markerSize` attribute sets the size of the marker in pixels. The default value is `5`.



---

### Attribute Example: markerFilled


```doenet-example
<graph size="small" displayXAxis="false" displayYAxis="false">
  <shortDescription>A graph of a filled marker and an open marker</shortDescription>
  <point coords="(-3,0)" markerStyle="square"><label>filled</label></point>
  <point coords="(3,0)" markerStyle="square" markerFilled="false"><label>open</label></point>
</graph>
```



The `markerFilled` attribute determines whether the marker is drawn solid (`true`) or as an open outline (`false`). The default value is `true`. It has no effect when `markerStyle` is `cross` or `plus`, which have no interior.



---

### Attribute Example: Standard graphical attributes


```doenet-editor-horiz
<graph size="small">
  <shortDescription>A graph of two styled points and a circle</shortDescription>
  <circle styleNumber="5" filled radius="6" fixed/>
  <point 
    name="A" layer="1" 
    styleNumber="3" 
    applyStyleToLabel 
    labelIsName 
    hide="$hidePointA"
  />
  <point 
    name="B" 
    applyStyleToLabel 
    styleNumber="2" 
    maskLabel="true"
    labelIsName coords="(3,3)"
  />
</graph>

<booleanInput name="hidePointA"><label>Hide pointA</label></booleanInput>
<p>Standard graphical attributes:</p>
    <ul>
      <li><attr>hide</attr></li>
      <li><attr>layer</attr> (0) is default value.</li>
      <li><attr>styleNumber</attr> 1 - 6</li>
      <li><attr>applyStyleToLabel</attr></li>
      <li><attr>maskLabel</attr></li>
      <li><attr>labelIsName</attr></li>
    </ul>
```



The uses of the following standard graphical attributes are illustrated: `hide`, `layer`, `styleNumber`, `applyStyleToLabel`, `maskLabel` and `labelIsName`.

By default, a point's label is drawn with a transparent background; setting `maskLabel="true"` (shown on point `B`) gives the label an opaque background so it stays legible over an axis or grid line. See the [`<label>{:dn}`](label) reference for a side-by-side comparison.



## Property Examples

### Property Example: Attributes as properties


```doenet-editor-horiz
<graph size="small">
  <shortDescription>A graph of a point labeled p</shortDescription>
  <point name="p" labelIsName>(2,3)</point>
</graph>

<p><c>x, y</c> = $p.x , $p.y</p>
<p><c>coords</c> = $p.coords</p>
<p><c>xs</c> = $p.xs</p>
<p><c>labelPosition</c> = $p.labelPosition</p>
<p><c>showCoordsWhenDragging</c> = $p.showCoordsWhenDragging</p>
```



The above listed `<point>{:dn}` attributes are also accessible as properties.



---

### Property Example: numDimensions


```doenet-editor-horiz
<graph size="small">
  <shortDescription>A graph of a point labeled p</shortDescription>
  <point name="p" labelIsName>(5,5)</point>
</graph>

<p><c>numDimensions</c> = $p.numDimensions</p>
```



The `numDimensions` property renders the number of dimensions required to define the point.



---

### Property Example: latex


```doenet-editor-horiz
<graph size="small">
  <shortDescription>A graph of a point labeled p</shortDescription>
  <point name="p" labelIsName>(0,3)</point>
</graph>

<p><c>latex</c> = $p.latex</p>
```



The `latex` property renders the latex code associated with rendering the coordinates of the `<point>{:dn}`.



---

### Property Example: constraintUsed


```doenet-editor-horiz
<p>Move the point on and off the grid to observe changes to the property.</p>
<p><m>p = $p</m></p>

<p><c>constraintUsed</c> = $p.constraintUsed</p>

<graph size="small" grid="2 2">
  <shortDescription>A graph of a point with an attractToGrid constraint</shortDescription>
  <point name="p">(1.5,1.5)
    <attractToGrid dx="2" dy="2"/>
  </point>
</graph>
```



The `constraintUsed` property renders the boolean value corresponding to whether the current location of the `<point>{:dn}` is utilizing the specified constraint - in this case, an `<attractToGrid/>{:dn}` constraint.
