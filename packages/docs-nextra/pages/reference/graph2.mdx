### Attribute Example: description, decorative


```doenet-editor-horiz
<p>A graph without a description or decorative does not have enough information for assistive technology to interpret that graph and triggers a warning.</p>
<graph name="deficientGraph">
</graph>

<p>In general, each graph should have a description attribute set to a textual description of the graph contents.</p>
<graph><shortDescription>A graph of the unit circle</shortDescription>
  <circle fixed />
</graph>

<p>If for some reason, a graph is just present for decoration or contains no content that a user of a screen reader needs to be aware of, 
the attribute decorative should be added to the graph.</p>
<graph decorative />
```

Typically the `description` attribute should be specified for each ` <graph>{:dn}`.
In special cases, the `decorative` attribute could be specified instead to indicate it can be ignored by screen readers.

### Attribute Example: identicalAxisScales


```doenet-example
<p>Without <attr>identicalAxisScales</attr></p> 
<graph name="g" xmin="-5" xmax="5" grid="1 1" size="small">
  <shortDescription>A graph with different scales for the x and y axes</shortDescription>
  <point/>
  <line>y=x</line>
</graph>

<p>With <attr>identicalAxisScales</attr></p> 
<graph extend="$g" identicalAxisScales>
  <shortDescription>A graph that is tall so its axes are the same scale</shortDescription>
</graph>
```



When the `identicalAxisScales` attribute is used, the overall proportions of the ` <graph>{:dn}` will be proportional to the range of the $x$ and $y$ axes, and the grid will be perfectly square.

### Attribute Example: xLabelPosition, yLabelPosition


```doenet-example
<graph name="g" 
  xLabelPosition="left" 
  yLabelPosition="bottom" 
  showNavigation="false"
 ><shortDescription>A graph with the x-label at the left and the y-label at the bottom</shortDescription>
    <xLabel><m>x</m></xLabel>
    <yLabel><m>y</m></yLabel>
    <point/> 
    <line>y = x</line>
</graph>
```



Use the `xLabelPosition` attribute to choose between a `left` position and 
a `right` position for the $x$-axis label. The default position is `right`. 

Use the `yLabelPosition` attribute to choose between a `top` position and 
a `bottom` position for the $y$-axis label. The default position is `top`.

### Attribute Example: showNavigation


```doenet-example
<graph name="g" showNavigation="false" size="small"
 ><shortDescription>A graph where the navigation controls are not shown</shortDescription>
  <point/>
  <line>y = x</line>
</graph>

<graph extend="$g" showNavigation="true"
  <shortDescription>A graph where the navigation controls are shown (the default)</shortDescription>
</graph>
```



The `showNavigation` attribute controls whether the default zooming controls at the bottom of the ` <graph>{:dn}` are visible.
Even when the controls are not shown, the graph can still be panned by dragging it or zoomed by holding down Shift while scrolling.
By default, `showNavigation` is set to `true`.

### Attribute Example: fixAxes


```doenet-example
<graph name="g" fixAxes="true" size="small"
 ><shortDescription>A graph with the axes fixed</shortDescription>
  <point/>
  <line>y = x</line>
</graph>

<graph extend="$g" fixAxes="false" link="false"
  <shortDescription>A graph without the axes fixed (the default)</shortDescription>
</graph>
```



If the `fixAxes` attribute is set to `true`, the user cannot zoom in and 
out of the ` <graph>{:dn}` or pan the graph from side to side. The navigation buttons at the base of the graph 
will also not appear. By default, `fixAxes` is set to `false`.

### Attribute Example: grid


```doenet-editor-horiz
<p>Graph 1: If you zoom in and out below,
  the grid adapts.</p>
<graph name="g" grid size="small"
 ><shortDescription>A graph with a grid that adapts when the graph zoom is changed</shortDescription>
  <point/>
  <line>y = x</line>
</graph>

<p>Graph 2: If you zoom in and out below, 
  the grid remains constant.</p>
<graph extend="$g" grid="1 1"
  <shortDescription>A graph with a grid that is fixed at interval of 1</shortDescription>
</graph>

<p>Graph 3: If you zoom in and out below, 
  the grid remains constant.</p>
<graph extend="$g" grid="5 1"
  <shortDescription>A graph with a grid that is fixed at an interval 5 in the x-direction and 1 in the y-direction.</shortDescription>
</graph>

<p>Graph 4: If you zoom in and out below, 
  the grid adapts.</p>
<graph extend="$g" grid="dense"
  <shortDescription>A graph with a dense grid that adapts when the graph zoom is changed</shortDescription>
</graph>
```

The `grid` attribute specifies the spacing of the gridlines. Valid values are `none`, `medium`, `dense`, 
and `a b` (where a is the x-axis spacing and b is the y-axis spacing).

The default ` <graph>{:dn}` does not show a grid; however, if only `grid` is specified, 
then a default medium grid is shown.

Unless the `grid="a b"` specification is used, all grid spacings adapt to the scale of the graph.

### Attribute Example: showBorder


```doenet-editor-horiz
<graph 
  displayXAxis="false" 
  displayYAxis="false" 
  showNavigation="false" 
  showBorder="false"
 ><shortDescription>A graph without a border showing a pentagon, circle, and rectangle</shortDescription>
    <regularPolygon 
      numSides="5" 
      sideLength="4" 
      center="(0,4)" 
      filled/>
    <circle 
      center="(-4,-4)" 
      radius="3" 
      styleNumber="2" 
      filled/>
    <rectangle 
      center="(4,-4)" 
      width="5" 
      height="3" 
      styleNumber="3" 
      filled/>
</graph>
```



The `showBorder` attribute can be set to `false` to suppress rendering of the default border.

### Property Example: xScale, yScale


