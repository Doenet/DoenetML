# ` <problem>{:dn}`



` <problem>{:dn}` renders an auto-numbered block of content. Inline components such as paragraphs (` <p>{:dn}`) and other components can be nested within an ` <problem>{:dn}` to group and order content. Some examples are: another ` <problem>{:dn}`, a ` <section>{:dn}`, a ` <figure>{:dn}`, or an ` <example>{:dn}`.


---

### Example: ` <problem>{:dn}` with default title


```doenet-editor-horiz
<problem>
  <p><answer>42</answer></p>
</problem>
<section>
  <p><lorem generateWords="10"/></p>
</section>
<exercise>
  <p>Build a water rocket.</p>
</exercise>
```



Within a DoenetML document, default titles for sectional components are auto-numbered. Each sectional component is on the same counter.



---

### Example: ` <problem>{:dn}` with custom title


```doenet-editor-horiz
<problem><title>Factor the trinomial</title>
  <p><m>x^2 + 3x + 2 = </m>
    <answer symbolicEquality>
      <award>(x+2)(x+1)</award>
      <award>(x+1)(x+2)</award>
    </answer>
  </p>
</problem>
```



A custom ` <problem>{:dn}` title is creating by providing a ` <title>{:dn}` tag within the ` <problem>{:dn}`.



---

### Example: Nesting sectional components


```doenet-editor-horiz
<problem><title>This is the main problem.</title>
  <p>Move the point.</p>
  <graph size="tiny">
    <point name="p"/>
  </graph>
  <answer>
    <award><when>$p != (0,0)</when></award>
  </answer>

---

    <p><lorem generateWords="10"/></p>


</problem>
```



A hierarchy is created by nesting one or more sectional components within an ` <problem>{:dn}`.



---

### Example: newNameSpace


```doenet-editor-horiz
<problem boxed>
  <p>This is the same namespace as the larger document.</p>
  <number name="x">sqrt(2)</number>  
  <p><m>x = $x</m></p>
</problem>
<problem newNameSpace name="secondProblem" boxed>
  <p>This is a separate, distinct namespace.</p>
  <number name="x">3/4</number>  
  <p><m>x = $x</m></p>
</problem>  
<p>General document: <m>x = $x</m></p>
<p>Second problem: <m>x</m> = $(secondProblem/x)</p>
```



The `newNameSpace` attribute defines a separate namespace within the sectional component. Values corresponding to that namespace can be accessed outside the namespace by prepending the name of the problem.



---

### Example: includeParentNumber


```doenet-editor-horiz
<section boxed>
  <problem>
    <p>This problem is nested inside a section. The default number does not include the parent number.</p>
  </problem>
</section>
<section boxed>
  <problem includeParentNumber="true">
    <p>This exercise is nested inside a section. Rendering of the parent section number has been included.</p>
  </problem>
</section>
```



The `includeParentNumber` attribute is `false` by default for the ` <problem>{:dn}` component (and `true` by default for the ` <section>{:dn}` component).


In order to include the number of the parent sectional component, set `includeParentNumber` to `true`.



---

### Example: includeAutoNumber


```doenet-editor-horiz
  <problem boxed><title>Custom title</title>
    <p>In this problem, the custom title overrides the default numbering of the problem.</p>
  </problem>
  <problem boxed includeAutoNumber="true">
### Example: Special title

    <p>In this problem, the number of the problem is still rendered in addition to the custom title.</p>
  </problem>
```



The `includeAutoNumber` attribute is `false` by default for the ` <problem>{:dn}` component. This means that if a problem contains a nested ` <title>{:dn}`, it will not render the default numbering (although it will still be counted in the numbering scheme behind the scenes).


In order to display the problem number in addition to the custom ` <title>{:dn}`, set the `includeAutoNumber` attribute to `true`.



---

### Example: aggregateScores and sectionWideCheckWork


```doenet-editor-horiz
<problem sectionWideCheckWork aggregateScores><title>Multiple questions</title>
  <p>Enter <m>x</m>: <answer>x</answer></p>
  <p>Enter <m>y</m>: <answer>y</answer></p>
  <p>Enter <m>z</m>: <answer>z</answer></p>
</problem>
```



The `aggregateScores` and `sectionWideCheckWork` attributes together combine validation for all ` <answer>{:dn}` components within the ` <problem>{:dn}` under a single "Check Work" button.

### Example: boxed


```doenet-editor-horiz
<problem boxed><title>Boxed problem</title>
<p>Move the point.</p>
  <graph size="tiny">
    <point name="p"/>
  </graph>
  <answer>
    <award><when>$p != (0,0)</when></award>
  </answer>  
</problem>
```



The `boxed` attribute changes the default formatting for a sectional component to one that has a border and a shaded banner for the title.



---

### Example: hide


```doenet-editor-horiz
<booleanInput name="hideProblem">
  <label>Click here to hide the problem</label>
</booleanInput>
<problem hide="$hideProblem"><title>Now you see me...</title>
  <p>Type "sassafras" <answer type="text">sassafras</answer>.</p>
</problem>
```



The `hide` attribute takes a boolean as input and can be used with any rendered component.
