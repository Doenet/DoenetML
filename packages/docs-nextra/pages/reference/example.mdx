# ` <example>{:dn}`


` <example>{:dn}` renders an auto-numbered block of content. Inline components such as paragraphs (` <p>{:dn}`) and other components can be nested within an ` <example>{:dn}` to group and order content. Some examples are: another ` <example>{:dn}`, a ` <section>{:dn}`, a ` <figure>{:dn}`, or a ` <problem>{:dn}`.

### Example: ` <example>{:dn}` with default title


```doenet-editor-horiz

---

  <p>Interactive activity here.</p>


<section>
  <p><lorem generateWords="10"/></p>
</section>
<problem>
  <p><answer>42</answer></p>
</problem>
```



Within a DoenetML document, default titles for sectional components are auto-numbered. Each sectional component is on the same counter.

### Example: Nesting sectional components


```doenet-editor-horiz

---

  <p><lorem generateWords="10"/></p>
  <exercise><title>This is an exercise within an example</title>
    <p><lorem generateWords="10"/></p>
  </exercise>


```



A hierarchy is created by nesting one or more sectional components within an ` <example>{:dn}`.

### Example: newNameSpace


```doenet-editor-horiz

---

  <p>This is the same namespace as the larger document.</p>
  <number name="x">sqrt(2)</number>  
  <p><m>x = $x</m></p>



---

  <p>This is a separate, distinct namespace.</p>
  <number name="x">3/4</number>  
  <p><m>x = $x</m></p>


<p>General document: <m>x = $x</m></p>
<p>Second example: <m>x</m> = $(secondExample/x)</p>
```



The `newNameSpace` attribute defines a separate namespace within the sectional component. Values corresponding to that namespace can be accessed outside the namespace by prepending the name of the example.



---

### Example: includeParentNumber


```doenet-editor-horiz
<section boxed>

---

    <p>This example is nested inside a section. The default number does not include the parent section.</p>


</section>
<section boxed>

---

    <p>This example is nested inside a section. Rendering of the parent section number has been included.</p>


</section>
```



The `includeParentNumber` attribute is `false` by default for the ` <example>{:dn}` component (and `true` by default for the ` <section>{:dn}` component).


In order to include the number of the parent sectional component, set `includeParentNumber` to `true`.



---

### Example: includeAutoNumber


```doenet-editor-horiz

---

    <p>In this section, the custom title overrides the default numbering of the section.</p>



---

### Example: Special title

    <p>In this section, the number of the section is still rendered in addition to the custom title.</p>


```



The `includeAutoNumber` attribute is `false` by default for the ` <example>{:dn}` component. This means that if a section contains a nested ` <title>{:dn}`, it will not render the default numbering (although it will still be counted in the numbering scheme behind the scenes).


In order to display the example number in addition to the custom ` <title>{:dn}`, set the `includeAutoNumber` attribute to `true`.

### Example: aggregateScores and sectionWideCheckWork


```doenet-editor-horiz

---

  <p>Enter <m>x</m>: <answer>x</answer></p>
  <p>Enter <m>y</m>: <answer>y</answer></p>
  <p>Enter <m>z</m>: <answer>z</answer></p>


```



The `aggregateScores` and `sectionWideCheckWork` attributes together combine validation for all ` <answer>{:dn}` components within the ` <example>{:dn}` under a single "Check Work" button.
