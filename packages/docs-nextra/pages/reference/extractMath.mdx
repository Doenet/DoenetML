# ` <extractMath>{:dn}`


` <extractMath>{:dn}` is a [General Operator](../document_structure/essentialConcepts#component-types) 
component that extracts data from mathematical expresssions. (See also, the ` <matchesPattern>{:dn}` component 
for an alternative method for extracting data.)


The types of data that can be extracted with ` <extractMath>{:dn}` are: 

- the number of operands in the expression
- the individual operands themselves
- the functions involved in an expression
- the function arguments

### Example: Extracting operands and number of operands


```doenet-editor-horiz
<math name="m1">x^2 + 5</math>
<p>
  <extractMath type="operand" operandNumber="1">$m1</extractMath>
</p>
<p>
  <extractMath type="numOperands">$m1</extractMath>
</p>
```



To extract an `operand`, you must also specify the `operandNumber`.



---

### Example: Extracting functions and function arguments


```doenet-editor-horiz
<math name="m1" >f(x) + g(x,y)</math>
<p>Extract the function from the second operand:
  <extractMath type="function">
    <extractMath type="operand" operandNumber="2">$m1
    </extractMath>
  </extractMath>
</p>

<p>Extract the argument(s) of the function from the second operand:
  <extractMath type="functionArgument">
    <extractMath type="operand" operandNumber="2">$m1</extractMath>
  </extractMath>
</p>
```



To extract a `function` or `functionArgument`, you must first extract the `operand` under 
consideration. In order to extract an individual `functionArgument` in the case of a multivariable 
function; the `argumentNumber` must be provided as well.


Note that Doenet recognizes $f$ and $g$ as function symbols by default. If other symbols are to 
represent functions, they can be defined within the ` <math>{:dn}` component using the `functionSymbols` attribute.



---

### Attribute Example: type


```doenet-editor-horiz
<math name="m1">x + y + z</math>
<p>
  <extractMath type="numOperands">$m1</extractMath>
</p>    
```



The `type` attribute specifies the value to be extracted; it takes one of four possible 
inputs: `operand`, `numOperands`, `function`, and `functionArgument`.



---

### Attribute Example: operandNumber


```doenet-editor-horiz
<math name="m1">x + y + z</math>
<p>The third operand is:
  <extractMath type="operand" operandNumber="3">$m1</extractMath>
</p> 

<math name="m2">(x^2 - 1)/2 + sin(x)</math>
<p>The second operand is:
  <extractMath type="operand" operandNumber="2">$m2</extractMath>
</p>     
```



The `operandNumber` attribute is necessary when the `operand` type is extracted. It 
specifies the location of the requested operand within the expression.



---

### Attribute Example: argumentNumber


```doenet-editor-horiz
<math name="m1" >f(x) + g(x,y,z)</math>
<p>Extract the third argument of the function from the second operand:
  <extractMath type="functionArgument" argumentNumber="3">
    <extractMath type="operand" operandNumber="2">$m1</extractMath>
  </extractMath>
</p>
```



When function arguments are extracted from a math expression, the `argumentNumber` 
attribute specifies the location of the requested argument.
