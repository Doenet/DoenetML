# `<column>{:dn}` (in a matrix)

<ComponentDisplay name='column'/>

`<column>{:dn}` is a [Math](../document_structure/essentialConcepts#component-types)
component that defines the individual column entries of a [`<matrix>{:dn}`](matrix).
Inside a `<matrix>{:dn}`, the entries of each `<column>{:dn}` are math values
(rather than the [`<cell>{:dn}`](cell) children used in a
[`<spreadsheet>{:dn}`](spreadsheet)).

<AttrPropDisplay name='column'/>

## Examples

### Example: Defining a `<matrix>{:dn}` with `<column>{:dn}` components


```doenet-editor-horiz
<matrix name="I3">
  <column>1 0 0</column>
  <column>0 1 0</column>
  <column>0 0 1</column>
</matrix>
<p>I3.columns[3] = $I3.columns[3]</p>
```



One method for defining a `<matrix>{:dn}` in Doenet is to nest `<column>{:dn}`
components inside the opening and closing tags. Individual columns can be
accessed using index notation with the `columns` property of the `<matrix>{:dn}`.



---

### Example: A `<column>{:dn}` with symbolic math


```doenet-editor-horiz
<matrix name="A">
  <column>
    <math simplify>(f(x) + f(x))</math>
    <math>g(x)</math>
  </column>
  <column functionSymbols="h">g(x) h(x)</column>
</matrix>
<p>A.columns[2][2] = $A.columns[2][2]</p>
<p>A.columns[1] = $A.columns[1]</p>
```



Since a `<column>{:dn}` is a Math Component, some of the general attributes of a `<math>{:dn}`
also work inside a `<column>{:dn}`, such as `functionSymbols`. To use other `<math>{:dn}`
attributes (such as `simplify`), you can nest individual `<math>{:dn}` components within a `<column>{:dn}`.



## Attribute Examples

### Attribute Example: functionSymbols


```doenet-editor-horiz
<matrix name="A">
  <column functionSymbols="v x">x(t) v(t) a(t)</column>
  <column>x(t) v(t) a(t)</column>
</matrix>
<p>Column 1 uses <attr>functionSymbols</attr> = $A.columns[1]</p>
<p>Column 2 does not use this attribute = $A.columns[2]</p>
```



The `functionSymbols` attribute designates special characters or strings that are to be considered functions. By default, only the characters $f$ and $g$ are considered function symbols when employed within a Math Component.



---

### Attribute Example: splitSymbols


```doenet-editor-horiz
<p><m>A = </m>
  <matrix name="A">
    <column splitSymbols="false">
      ab ac
    </column>
    <column>
      ab ac
    </column>
  </matrix>
</p>
<p>Since column 2 defaults to <c>splitSymbols = true</c>, the sum of the entries <m>A_{12}</m> and <m>A_{22}</m> will simplify:</p>
<p><m>ab + ac  = a(b + c)</m> ? <nbsp/>
  <boolean>$A.columns[2][1] + $A.columns[2][2] = a(b + c)</boolean></p>
<p>However, the same is not true of column 1, since <m>A_{11}</m> is not equal to <m>a\cdot b</m>, and <m>A_{21}</m> is not equal to <m>a \cdot c</m>:</p>
<p><m>ab + ac  = a(b + c)</m> ? <nbsp/>
  <boolean>$A.columns[1][1] + $A.columns[1][2] = a(b + c)</boolean></p>
```



By default, letters placed adjacent to one another in a `<math>{:dn}` component are separated by Doenet for computations, so `xyz` is interpreted as $x \cdot y \cdot z$, rather than a single variable composed of three letters. To override this default behavior, set the splitSymbols attribute to false.


Since a `<column>{:dn}` is a type of Math Component, it follows this convention, with the exception that spaces between individual entries in a `<column>{:dn}` are considered distinct entries (similar to the entries in a `<mathList>{:dn}`, while in a general `<math>{:dn}`, a space represents implied multiplication.)



---

### Attribute Example: referencesAreFunctionSymbols


```doenet-editor-horiz
<select name="func" numToSelect="2">
  f g h a b q
</select>

<p><m>A = </m>
  <matrix name="A">
    <column
      referencesAreFunctionSymbols="$func[1] $func[2]">
      $func[1](x) $func[2](x)
    </column>
    <column>$func[1](x) $func[2](x)</column>
  </matrix>
</p>

<p>Column 1 uses <attr>referencesAreFunctionSymbols</attr> = $A.columns[1]
</p>

<p>Column 2 does not use this attribute = $A.columns[2]
</p>
```



The `referencesAreFunctionSymbols` attribute designates special characters or strings
that are to be considered functions, when they are references from another component by
name. In this case, the symbolic function letters are selected at random from a list of
6 possibilities.


Note that, even though `f` and `g` are default function symbols when written directly, in
this example they are referenced values and therefore we need to tell Doenet that they
represent functions.



## Property Examples

### Property Example: columns


```doenet-editor-horiz
<matrix name="I3">
  <column>1 0 0</column>
  <column>0 1 0</column>
  <column name="col3">0 0 1</column>
</matrix>

<p>I3.columns[3] = $I3.columns[3]</p>
<p>I3.columns[3][3] = $I3.columns[3][3]</p>
<p>col3 = $col3</p>
```



Technically speaking, the `columns` property belongs to the parent `<matrix>{:dn}`, rather than
the `<column>{:dn}`. The information is repeated here, however, because it is a useful way
of accessing individual columns and entries.
